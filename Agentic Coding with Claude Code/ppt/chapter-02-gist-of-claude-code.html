安裝 Claude Code

原生安裝程式

Claude Code 提供原生安裝程式，可自動偵測作業系統並安裝適當版本。

安裝步驟

  • 前往官方安裝指南：
    https://code.claude.com/docs/en/setup#native-install-recommended
  • 安裝程式會處理設定和更新
  • 支援多種作業系統

驗證安裝

claude --version

確認安裝成功後即可繼續設定 Next.js。

Claude Code 如何觀察專案

觀察機制

  • 掃描檔案系統：了解專案結構
  • 讀取程式碼：理解實作細節
  • 分析相依性：追蹤模組關係
  • 識別模式：發現程式碼慣例

環境建構

Claude Code 透過觀察專案自動建構環境，這個環境包含專案知識、架構決策和任何與特定專案相關的資訊。

環境的重要性

有意地管理環境對於獲得高品質且一致的結果至關重要。良好的環境讓 Claude 能夠：
  • 理解專案上下文
  • 遵循現有慣例
  • 做出一致的決策

Model Context Protocol (MCP)

什麼是 MCP？

MCP 提供標準化的方式，讓 AI 應用程式透過共同介面連接到工具、資料來源和服務。這是一個抽象層，啟用互通性、可移植性和生態系統成長。

MCP 的優勢

  • 一次實作，多次使用：在不同客戶端重複使用
  • 標準化介面：跨平台相容性
  • 生態系統成長：社群持續貢獻新伺服器
  • 可擴展性：輕鬆添加新功能

實際應用

  • 連接 Playwright MCP 進行測試
  • 存取外部 API 和服務
  • 擴充 Claude Code 的原生能力
  • 整合工作流程到日常開發

權限系統

為什麼需要權限？

Claude Code 需要明確的權限來執行各種操作。權限系統確保安全性，讓使用者可以精確控制 Claude 可以執行的動作。

權限類型

  • 檔案讀取：存取專案檔案
  • 檔案寫入：建立和修改檔案
  • 指令執行：執行終端機指令
  • 網路存取：調用外部 API

權限控制

  • 明確批准每個動作
  • 全域和專案層級設定
  • 敏感操作需要額外確認
  • 可追蹤和審計權限使用
