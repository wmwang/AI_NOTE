Architecture // 指令階層

The 4-Tier Instruction Model

1. Project Instructions (GEMINI.md)
Team-shared, committed to repo. (團隊共享，提交至倉庫)
2. Subdirectory Instructions (src/GEMINI.md)
Scoped rules for specific modules. (特定模組的局部規則)
3. Private Project Memory (MEMORY.md)
Local setup, machine-specific notes (NOT committed). (本地設置，私有筆記)
4. Global Personal Memory (Global GEMINI.md)
Personal preferences across all projects. (跨專案的全局偏好)

Tier 1 // 專案層級

GEMINI.md: The Team Contract

This file lives in the root and is visible to Everyone on the team.

# Project Rules
- Use `pnpm` only.
- Always use TypeScript strict mode.
- Commit messages must follow Conventional Commits.
- Pre-requisite for PR: Run `npm run lint`.
Best Practice

Treat GEMINI.md as the Living Documentation of your project's engineering culture.

Subdirectory GEMINI.md

Fact Routing: Where to save?

Team convention?

➡️ GEMINI.md

Machine-specific setup?

➡️ MEMORY.md

Personal coding style?

➡️ Global GEMINI.md

Narrow module rule?

➡️ Subdir GEMINI.md

"Never duplicate facts across tiers. One fact, one home."

Hygiene // 記憶維護

Memory Hygiene

Don't let your memory files become a "Dumpster". (不要讓記憶文件變成垃圾堆。)

BAD ❌

Saving transient session state or temporary bug fixes in GEMINI.md.

GOOD ✅

Saving long-lived project context and architectural principles.
