Operations // 操作

Resolving Issues by ID

Pointing Claude at an issue allows it to focus on a specific problem statement. (將 Claude 指向 Issue 讓其專注於特定問題。)

$ claude /issue #42

Agentic Process:

  1. Reads issue description.
  2. Searches codebase for related symbols.
  3. Reproduces the bug (if possible).
  4. Proposes and applies the fix.

Memory // 記憶

The Repo Rulebook: CLAUDE.md

Ensure every agent session follows the same standards. (確保每個代理會話都遵循相同的標準。)

CLAUDE.md
# Coding Standards
- Use camelCase for variables.
- No `any` types allowed.
- Max file length: 300 lines.

# Architecture
- Components in `/ui`
- Logic in `/services`
Pro Tip

Commit CLAUDE.md to your repository so the entire team (and their agents) stays aligned.

Enforcing Standards Automatically

No more "nitpicking" in PR reviews. (PR 審核中不再需要挑骨頭。)

  • Agent checks CLAUDE.md before every commit.
    每次提交前檢查規則。

    確保遵循架構模式。
  • Automates the "boring" part of reviews.
    自動化審核中最無聊的部分。

"The agent is your first layer of Continuous Quality."

Automated Commit Messages

    節省開發者時間。

Safety // 安全性

Security in Automation

Rule 1: Never Solo-Merge

Agents should create PRs, but humans should always merge.

代理應該建立 PR，但人類應該負責合併。

Rule 2: Protect Secrets

Ensure `.env` and sensitive configs are in `.gitignore` so the agent doesn't send them to the API.
