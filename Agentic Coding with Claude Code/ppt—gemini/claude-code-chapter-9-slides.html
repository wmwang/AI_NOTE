Definition // 定義

What are Agent Skills?

Skills are specialized packages of **Procedural Knowledge**.

Standard AI

"Fix this bug." (Depends on general LLM training)

Skilled AI

"Fix this bug using the *Enterprise-Security-Skill*." (Follows specific, expert-level procedures)

Skills turn the agent into a temporary "Domain Expert".

Logic // 程序化引導

Procedural vs. Declarative

Declarative (Instructions)

"Use Tailwind CSS."
"Keep code clean."

Tells the agent *What* to do.

Procedural (Skills)

"1. Read tailwind.config.js."
"2. Check for custom colors."
"3. Apply color-mix()."

Tells the agent *How* to do it.

Creation // 設計技能

How to Design a Great Skill

  • Identify a Pattern: Is this task repetitive and rule-based?
    識別模式：這項任務是否重複且基於規則？
  • Define Mandates: What MUST happen every time?
    定義授權：每次都必須發生什麼？
  • Provide Examples: Show what "Good" looks like.
    提供範例：展示什麼是「好」的。
  • Establish a Checklist: How do we verify success?
    建立清單：如何驗證成功？

Safety // 程序護欄

Procedural Guardrails

Skills prevent the agent from making "Stupid Mistakes".

Without Skill

Claude might delete a file to "Simplify" a solution, losing data.

With Skill

Skill mandate: "Never delete files. Rename to .old or move to /trash/."

Evolution // 技能演進

Iterating on Skills

Skills are not static. They evolve with your codebase.

Feedback: "Claude keeps using the wrong library for UI."
Action: Update the UI-Skill instructions with a "PROHIBITED" list.
Result: Future agents follow the new rule automatically.

Chapter 09 Key Takeaways

  • Skills provide expert, procedural guidance.
  • Activate them to solve complex, domain-specific problems.
  • Skills save context by being loaded only when needed.
  • Use `skill-creator` to codify your own best practices.
