Concept // 核心觀念

The Specialist Mindset

A senior engineer doesn't do everything manually. They delegate to experts.

👨‍💼
Main Agent (PM)

Maintains the big picture, manages the workspace.

🛠️
Subagents (Specialists)

Execute deep, narrow, or repetitive tasks.

Efficiency // 資源管理

Your Context Window is Precious

Every tool call and file read in the main thread consumes permanent history.

The Danger

"Context Drift" - The agent forgets the user's original goal because the history is full of search results.

The Fix

Move high-volume noise (searches, logs, big files) to a subagent. They return only a Summary.

Command // 如何使用

Tool: invoke_agent

The Main Agent calls this tool automatically when it identifies a high-impact task.

{
  "agent_name": "generalist",
  "prompt": "Find all instances where we use hardcoded URLs and move them to .env"
}

The prompt must be Comprehensive and Detailed. Subagents don't share the Main Agent's chat history.

Strategy // 委派策略 1

Handling High-Volume Output

Anti-pattern

Main agent runs npm test which outputs 5000 lines of failure logs directly into the history.

Proactive Move

Main agent asks a subagent to "Run the tests and summarize the top 3 unique failure patterns."

Strategy // 委派策略 2

Speculative Research

Sometimes you don't know the path until you try a few things.

Prompt: "I suspect a race condition in the buffer. Try to reproduce it with different delays and report back if you find a reliable trigger."

The subagent can fail 5 times silently and only return the 1 successful reproduction, keeping the main thread clean.

Safety // 並發與安全

The Concurrency Mandate

Multiple subagents can run, but Never on the same resource.

SAFE ✅

Subagent A researches UI.
Subagent B researches API.

DANGEROUS ❌

Subagent A edits `file.ts`.
Subagent B edits `file.ts` simultaneously.

Process // 運作流程

The Subagent Lifecycle

  1. Invocation: Main agent defines a clear goal.
  2. Independent Run: Subagent manages its own tools and turns.
  3. Termination: Subagent reaches a conclusion or error.
  4. Reporting: Subagent returns a structured summary to the Main Agent.

Quality // 質量控制

Trust but Verify

Don't assume the subagent's summary is 100% correct. (不要假設子代理的總結完全正確。)

  • Cross-check key file paths.
  • Ask the subagent to show excerpts.
  • Main agent should perform a "Final Check" before committing code.
Pro Tip

If a subagent fails, analyze its final report to see if the Prompt was too vague.

Avoid // 避免誤區

Pitfall: Over-delegation

Delegation has overhead. Don't use subagents for simple things.

  • Simple Reads: Just use `read_file` in the main thread.
  • Single-line Edits: `replace` is faster than invoking a subagent.
  • Quick Questions: If you can answer it in 1 turn, don't delegate.

Advanced: Swarm Orchestration

The Investigation Swarm

Main Agent spawns 3 investigators to map 3 different modules simultaneously. Gathers all reports into a unified SPEC.md.

The Cleanup Swarm

Main Agent spawns a generalist to fix tests and another to fix linting in parallel folders.

Conclusion // 總結

Chapter 07 Key Takeaways

  • Subagents are for **Complexity** and **Volume**.
  • Use them to keep your Main Agent's context lean and focused.
  • Comprehensive prompts are key for successful delegation.
  • Never run subagents that mutate the same files concurrently.
