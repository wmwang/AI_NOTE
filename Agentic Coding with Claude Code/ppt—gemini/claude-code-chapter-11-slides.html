Concept // 核心觀念

Fast vs. Slow Thinking

Standard Agent (Fast)

Reactive. Fixes local issues. "What can I do now?"

Great for: Bug fixes, small features, refactoring.

Deep Agent (Slow)

Proactive. Explores architecture. "Why is it built this way?"

Great for: Planning, auditing, major migrations.

Mechanism // 運作機制

The Reasoning Loop

Deep Agents don't just act; they build a **World Model** first.

Hypothesize
➡️
Investigate
➡️
Validate

If validation fails, the loop restarts with a new hypothesis. (如果驗證失敗，循環將帶著新假設重新開始。)

Investigation // 遞歸搜尋

Recursive Investigation

Deep Agents follow the breadcrumbs across the entire stack.

1. Finds usage in UI.
2. Traces to API Hook.
3. Traces to Backend Controller.
4. Traces to Database Schema.
5. Identifies mismatch in data types.
The Depth

Standard agents usually stop at step 2. Deep agents won't stop until they reach the **Source of Truth**.

Quality // 自我修正

The Power of Self-Correction

Deep Agents are taught to be their own Harshest Critics.

➜ Thinking: My previous assumption about the cache was wrong. The latency persists even with cache disabled.
➜ New Hypothesis: The bottleneck is in the connection pool. I will now run a stress test to confirm.

Strategy // 應用場景

When to go "Deep"?

YES ✅
  • "Audit our security layer for potential leaks."
  • "Why is the build 2x slower this week?"
  • "Design a migration path to Microservices."
NO ❌
  • "Change this button color to blue."
  • "Fix this typo."
  • "Add a unit test for this function."

Philosophy // 耐心是關鍵

Patience is a Feature

Reasoning takes time. Deep sessions can last for Minutes, not seconds.

"Speed is the enemy of Depth."

If you want a Deep Agent to succeed, you must give it the space to 'Think' without interruption.

Cost // 資源管理

Managing the High Cost

Deep reasoning is token-heavy and compute-intensive.

Token Budgets

Deep sessions can easily consume 50k+ tokens in reasoning alone.

Rate Limits

Be aware of your organization's API tier when running multiple deep agents.

The Deep Analysis Report

Deep Agents produce high-density artifacts instead of just code edits.

## Root Cause Analysis: System Latency
- **Primary Cause**: N+1 query pattern in `UserService`.
- **Secondary Cause**: Inefficient index on `Orders` table.
- **Evidence**: 15 trace logs attached.
- **Recommendation**: Implement `DataLoader` pattern and re-index.

The "Ghost in the Machine" Bug

Issue: Intermittent data corruption in production.

Standard Agent: "Can't reproduce. Logs look fine."

Deep Agent: Found a race condition in a 3rd-party library used by a dependency of a dependency.

The agent analyzed the raw source of the node_module and found a non-thread-safe singleton.

Guidelines // 互動指南

How to talk to a Deep Agent

  • Give it a "Long Leash": Don't micromanage the first 5 minutes.
  • Provide "Seed Knowledge": Point it to the most relevant entry point.
  • Challenge its Assumptions: "Are you sure the issue isn't in module Y?"
