# 練習 3：Prompt 工程 — 改造你的 Prompt

> **時間：10 分鐘**
> **模組：M3 — Context Engineering**

---

## 目標

拿出 Ex2 你用過的 Prompt（或使用下方提供的「待改造版本」），
用 Context Engineering 三要素重新設計，比較輸出品質的差異。

---

## 待改造的 Prompt（原始版）

```
幫我寫一個 TypeScript 函式，可以計算訂單折扣。
折扣碼 SAVE10 打九折，SAVE15 打八五折。
Gold 會員也打九折。
```

**這個 Prompt 的問題：**
- 沒有角色：AI 不知道用什麼標準思考
- 沒有格式要求：輸出形式不可預測
- 沒有測試要求：不保證函式正確性
- 沒有邊界說明：AI 會自己「腦補」

---

## Step 1：分析並填寫（3 分鐘）

**角色（Role）：**
這個 Prompt 需要一個什麼樣的 AI 角色？

> 我需要 AI 扮演：____________________________________

**格式（Format）：**
我希望 AI 輸出什麼格式？（勾選）

- [ ] TypeScript 函式（有型別定義）
- [ ] 搭配 Jest 測試
- [ ] 只有函式，不要解釋
- [ ] 加上 JSDoc 說明
- [ ] 其他：______________

**規則（Rules）：**
有哪些限制或要求？（寫出 3 條以上）

1. ___________________________________________________
2. ___________________________________________________
3. ___________________________________________________
4. （選填）___________________________________________

**範例（Examples）：**
你能提供一個 few-shot 範例嗎？

```
輸入：[你的範例輸入]
輸出：[你期望的輸出格式]
```

---

## Step 2：重新設計 Prompt（4 分鐘）

根據上面的分析，完成改造後的 Prompt：

```
[角色]
你是一位 ________________________________________，
擅長 ________________________________________________。

[任務]
請根據以下需求，生成 ________________________________。

[規則]
- 規則 1：_____________________________________________
- 規則 2：_____________________________________________
- 規則 3：_____________________________________________

[格式/範例]（選填）
格式參考：
  輸入：____________
  輸出：____________

[需求]
折扣計算規則：
- 折扣碼：SAVE10 → 0.9, SAVE15 → 0.85, VIP20 → 0.8
- 會員等級：Gold → 0.9, Silver → 0.95, Bronze → 1.0
- 取折扣較大者
- 無效折扣碼拋出 "INVALID_DISCOUNT_CODE"
- 空訂單拋出 "EMPTY_ORDER"
```

---

## Step 3：測試改造效果（3 分鐘）

1. 把**改造後**的 Prompt 貼給 AI
2. 把**原始版** Prompt 的輸出（或重新問一次）和改造後的比較

**記錄比較：**

| | 原始 Prompt | 改造後 Prompt |
|--|------------|--------------|
| 輸出有型別定義？ | ☐ 有 ☐ 沒有 | ☐ 有 ☐ 沒有 |
| 有測試？ | ☐ 有 ☐ 沒有 | ☐ 有 ☐ 沒有 |
| 處理了錯誤情況？ | ☐ 有 ☐ 沒有 | ☐ 有 ☐ 沒有 |
| 你需要修改幾次？ | ___ 次 | ___ 次 |
| 整體品質（1-5）| ___ | ___ |

## Bonus：設計一個可複用的 Prompt 模板

如果你完成了基本練習，試著把改造後的 Prompt 轉換成「模板格式」：

```markdown
# Prompt 模板：[功能名稱]

## 用途
[描述這個 Prompt 的用途]

## Prompt
---
[角色]
[任務]
[規則]
[格式]

[輸入]：{{INPUT}}
---

## 使用說明
- 把 {{INPUT}} 替換成 [說明]
- 最適合用在 [使用情境]
- 注意事項：[任何特殊注意事項]
```

**想一想：** 你的團隊裡有沒有其他常見的 AI 使用情境，也可以設計成模板？
（例如：Code Review、API 設計、DB Schema 設計、錯誤訊息改善...）
