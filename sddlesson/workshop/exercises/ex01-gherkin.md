# 練習 1：撰寫你的第一個 Gherkin

> **時間：10 分鐘**
> **模組：M1 — 規格先行**

---

## 情境說明

你是某電商平台的工程師，PO 給了你這個需求：

> 「使用者可以搜尋商品。搜尋結果要依相關度排序，
>  可以依價格、評分篩選。沒有結果要顯示提示。」

---

## Step 1：Discovery 三問（3 分鐘）

填入以下空格：

**Actor（誰在使用這個功能）：**
> ____________________

**Context（在什麼前提下使用）：**
> ____________________

**Outcomes（可能的結果有哪些？）：**

| 情境 | 結果 |
|------|------|
| 搜尋關鍵字有結果 | ______________________ |
| 搜尋關鍵字無結果 | ______________________ |
| 關鍵字為空 | ______________________ |
| ？（你自己想到的）| ______________________ |

---

## Step 2：撰寫 DSL-Level Gherkin（7 分鐘）

請寫出 **3 個** Scenario：
- 1 個 Happy Path（主流程，搜尋成功有結果）
- 2 個 Edge Case（從 Discovery 的邊界情況中選）

```gherkin
Feature: 商品搜尋

  Background:
    Given 商品資料庫中有以下商品：
      | 名稱       | 價格  | 評分 | 類別   |
      | iPhone 殼 | 250   | 4.5  | 手機配件 |
      | 藍芽耳機  | 1200  | 4.8  | 耳機   |
      | 充電線    | 80    | 3.9  | 配件   |

  Scenario: [你的 Happy Path 名稱]
    Given [前置條件]
    When  [使用者的動作]
    Then  [預期結果]
    And   [另一個預期結果（如果有）]

  Scenario: [Edge Case 1 名稱]
    Given [前置條件]
    When  [使用者的動作]
    Then  [預期結果]

  Scenario: [Edge Case 2 名稱]
    Given [前置條件]
    When  [使用者的動作]
    Then  [預期結果]
```

---

## 評分標準（自我檢核）

完成後，用以下標準檢查你的 Gherkin：

- [ ] Given 描述的是「狀態」，不是「使用者操作」
- [ ] When 只有一個核心動作（不要把兩件事合在一起）
- [ ] Then 描述的是可觀察的結果（不是實作細節）
- [ ] 每個 Scenario 可以獨立閱讀、獨立理解
- [ ] Edge Case 是真實可能發生的（不是故意刁難）

---

## 提示（如果卡住了）

**Happy Path 提示：**
使用者在搜尋欄輸入「藍芽耳機」，應該要看到什麼？

**Edge Case 1 提示：**
如果使用者搜尋「xxx不存在的商品」會怎樣？

**Edge Case 2 提示：**
如果使用者什麼都不輸入，直接按搜尋會怎樣？

**Advanced（選做）：**
如果你完成了基本要求，試著寫一個「篩選條件」的 Scenario：
使用者搜尋「配件」，然後設定「價格低於 $200」的篩選。
