# Ollama 的 Open WebUI

Open WebUI（前身為 Ollama WebUI）是一個功能豐富、可自託管的網頁介面，用於運行與各種 LLM 運行器（包括 Ollama 和相容 OpenAI 的 API）的對話。它提供了一種使用者友善的方式來互動本地語言模型。

## 為什麼使用 Open WebUI？

- **使用者友善介面**：現代化的、類似 ChatGPT 的聊天介面
- **自託管**：完全控制您的資料和部署
- **多模型支援**：與 Ollama 和 OpenAI 相容的 API 搭配使用
- **豐富功能**：文件上傳、RAG、標記、歷史記錄等
- **多使用者支援**：多用戶存取的身份驗證和權限
- **響應式設計**：在桌面和行動裝置上運作良好

## 安裝

### Docker 安裝（推薦）

如果您已安裝 Ollama，最簡單的入門方式是使用 Docker：

```bash
docker run -d -p 3000:8080 \
  --add-host=host.docker.internal:host-gateway \
  -v open-webui:/app/backend/data \
  --name open-webui \
  --restart always \
  ghcr.io/open-webui/open-webui:main
```

### 如果 Ollama 在您的電腦上

如果您使用原生安裝的 Ollama 運行：

```bash
docker run -d -p 3000:8080 \
  -e OLLAMA_BASE_URL=http://host.docker.internal:11434 \
  -v open-webui:/app/backend/data \
  --name open-webui \
  --restart always \
  ghcr.io/open-webui/open-webui:main
```

### 使用 Docker Compose

創建一個 `docker-compose.yml`：

```yaml
services:
  ollama:
    image: ollama/ollama
    ports:
      - "11434:11434"
    volumes:
      - ollama:/root/.ollama

  open-webui:
    image: ghcr.io/open-webui/open-webui:main
    ports:
      - "3000:8080"
    environment:
      - OLLAMA_BASE_URL=http://ollama:11434
    volumes:
      - open-webui:/app/backend/data
    depends_on:
      - ollama

volumes:
  ollama:
  open-webui:
```

然後運行：

```bash
docker compose up -d
```

### Python pip 安裝

```bash
pip install open-webui
open-webui serve
```

## 初始設定

1. **訪問 WebUI**：開啟瀏覽器並導航到 `http://localhost:3000`
2. **建立管理員帳號**：第一位註冊的使用者會自動成為管理員
3. **配置 Ollama 連接**：WebUI 通常會自動偵測在 `localhost:11434` 的 Ollama
4. **下載模型**：使用管理員設定來拉取模型或透過 CLI 進行

### 管理員設定

作為管理員，您可以配置：

- **模型管理**：下載、刪除和管理模型
- **使用者管理**：新增使用者、設定角色、管理權限
- **系統設定**：配置預設值、安全性、功能
- **Ollama 設定**：連接參數和資源使用
- **API 金鑰**：管理 OpenAI 和其他外部 API 的存取

## 核心功能

### 聊天介面

主要的聊天介面提供：

- **模型選擇**：在對話之間切換模型
- **系統提示**：為每個對話設定自訂指令
- **溫度調節**：調整回應的創造力
- **串流回應**：即時觀看回應的生成
- **複製/重新生成**：輕鬆複製回應或嘗試替代方案

### 文件上傳與 RAG

Open WebUI 支援檢索增強生成（RAG）：

1. **上傳文件**：支援 PDF、TXT、DOCX、CSV 等
2. **自動處理**：文件被分塊並嵌入以供檢索
3. **查詢文件**：在聊天中提問時引用相關內容
4. **引用來源**：查看使用了哪些文件區塊

### 模型管理

- **拉取模型**：直接從 Ollama 庫下載模型
- **模型資訊**：查看參數大小、修改日期
- **批量操作**：一次性拉取或刪除多個模型
- **模型標記**：組織和分類模型

### 對話管理

- **對話歷史**：所有對話都被保存
- **搜尋**：在對話歷史中搜尋
- **資料夾**：將對話組織到資料夾中
- **標記/釘選**：將重要對話加入書籤
- **匯出**：匯出對話以供分享或備份

## 進階功能

### 自訂模型（Modelfiles）

直接透過 WebUI 創建自訂模型：

1. 導航到 **工作區 → 模型**
2. 點擊 **+** 建立新模型
3. 設定基礎模型、系統提示和參數
4. 儲存並立即開始使用

### OpenAI API 整合

Open WebUI 可以連接到 OpenAI 以及相容的 API：

1. 前往 **設定 → 連接**
2. 新增您的 OpenAI API 金鑰
3. 配置 API 端點 URL
4. 在 Ollama 模型旁邊使用 OpenAI 模型

### 函數呼叫（工具）

建立自訂函數以擴展功能：

- 網路搜尋整合
- 自訂 API 呼叫
- 資料庫查詢
- 程式碼執行

### 社群整合

使用內建的整合功能連接到外部工具：

- **文件編輯器**：直接在瀏覽器中編輯文件
- **圖像生成**：整合 Stable Diffusion 或 DALL-E
- **語音**：語音轉文字輸入
- **網路搜尋**：在聊天中搜尋網路

## 配置選項

### 環境變數

關鍵的環境變數包括：

| 變數 | 描述 | 預設值 |
|----------|-------------|---------------|
| `OLLAMA_BASE_URL` | Ollama API 端點 | `http://localhost:11434` |
| `OPENAI_API_BASE_URL` | OpenAI API 端點 | `https://api.openai.com/v1` |
| `OPENAI_API_KEY` | OpenAI API 金鑰 | - |
| `WEBUI_SECRET_KEY` | 工作階段加密金鑰 | 自動生成 |
| `DATA_DIR` | 資料儲存目錄 | `/app/backend/data` |
| `ENABLE_SIGNUP` | 啟用使用者註冊 | `true` |
| `DEFAULT_USER_ROLE` | 新使用者的角色 | `pending` |

### 效能調整

- **Ollama 設定**：配置並行請求和超時
- **嵌入模型**：選擇 RAG 的嵌入模型
- **分塊設定**：調整文件處理參數
- **記憶體限制**：為 Docker 容器設定資源限制

## 安全性考量

- **身份驗證**：始終為面向公網的部署設定管理員密碼
- **HTTPS**：在生產環境中使用反向代理（nginx）處理 SSL
- **網路**：限制對信任網路的存取
- **更新**：保持 Open WebUI 映像檔更新
- **備份**：定期備份資料卷

## 故障排除

### 常見問題

**無法連接到 Ollama**
- 確保 Ollama 正在運行：`ollama list`
- 檢查 Ollama 是否在預設埠（11434）上
- 驗證 Docker 網路配置

**模型未出現**
- 確保模型已拉取：`ollama pull <model-name>`
- 在設定中檢查 Ollama 連接
- 重新啟動 Open WebUI 容器

**文件上傳問題**
- 檢查 Docker 卷權限
- 確保有足夠的磁碟空間
- 驗證檔案大小限制

## 總結

Open WebUI 為 Ollama 和相容模型提供了一個優秀的、功能豐富的前端。它的自託管特性確保了資料隱私，同時提供了與商用解決方案相當的功能。對於希望擁有私人、可自訂的 AI 聊天介面的開發者和團隊來說，它是理想的選擇。
