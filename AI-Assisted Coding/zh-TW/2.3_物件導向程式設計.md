# 2.3 物件導向程式設計

將程式碼結構化為函式或方法只是朝著深思熟慮地開發複雜大型程式的第一步。更困難的是以有意義的物件導向方式設計程式碼。什麼屬於一個類別？哪些資料是公開的，哪些是私有的？建構子必須處理哪些細節？在哪些（資料）邊界上類別相互區分？在哪裡可以適當地使用繼承或介面等機制？

與前面的章節一樣，我們這裡的目的不是教您如何程式設計。我們也不認為在這裡印出大量程式碼是合適的。我們的首要目標是透過範例激勵您讓 AI 機器人或精靈為您提供有針對性的支援。這不僅適用於程式碼的撰寫，而且——更重要的是——適用於類別的正確設計。

> **為什麼選擇 Java？**
> 以下範例使用 Java 作為程式語言。雖然您也可以在 Python 中定義類別，但其優勢在於其他方面。Java 提供了更多的物件導向功能。
>
> 誠然，Java 是一種相當老的語言。您可以在 Swift 或 Kotlin 等新程式語言中更優雅地撰寫物件導向程式碼。然而，這些語言相對較新，功能會更改或新增。由於這個原因，以及有限的分佈（即有限的訓練材料可用性），語言模型對於歷史悠久的語言比對於新語言運作得更好。
>
> 長話短說——即使我們對 Java 的熱愛有限，這種廣泛使用的語言似乎是以下範例的最佳選擇。

## 2.3.1 範例：使用者帳戶類別

以下提示旨在展示 AI 工具如何幫助您進行程式的物件導向設計。第一個提示涉及一個程式區段，其中使用者帳戶由一個 Java 類別表示。在提示中，我們總結了類別的目標，列出了其最重要的屬性，並命名了我們使用的程式語言。然後才是實際的問題：類別的建構子應該執行什麼樣的驗證？

**提示：** I need to write a Java class for a user account. Important attributes are account name, full name, contact email, password hash, and a list of recent login times. What kind of validation should I do in the constructor?

> 在建立具有帳戶名稱、全名、聯絡電子郵件、密碼雜湊和最近登入時間列表等屬性的使用者帳戶 Java 類別時，包含適當的驗證以確保資料完整性和安全性非常重要。以下是建議的 Java 類別及所需的驗證：
> ```java
> import ...;
> public class UserAccount {
>     private String accountName;
>     private String fullName;
>     private String contactEmail;
>     private String passwordHash;
>     private List<String> recentLoginTimes;
>     // 建構子
>     public UserAccount(String accountName, ...) {
>         this.setAccountName(accountName);
>         this.setFullName(fullName);
>         ...
>     }
>     // 建構子使用的驗證方法
>     private void setAccountName(String accountName) {
>         if (accountName == null ||
>             accountName.isEmpty() ||
>             accountName.length() < 3 ||
>             accountName.length() > 20)
>         {
>             throw new IllegalArgumentException(
>               "Name must be between 3 and 20 characters.");
>         }
>         this.accountName = accountName;
>     }
>     private void setFullName(String fullName) { ... }
>     private void setContactEmail(String contactEmail) { ... }
>     private void setPasswordHash(String passwordHash) { ... }
>     private void setRecentLoginTimes(
>                        List<String> recentLoginTimes) { ... }
>     // 驗證電子郵件格式的工具方法
>     private boolean isValidEmail(String email) { ... }
>     // Getters
>     public String getAccountName() { return accountName; }
>     public String getFullName() { ... }
>     public String getContactEmail() { ... }
>     public String getPasswordHash() { ... }
>     public List<String> getRecentLoginTimes() { ... }
> }
> ```
>
> **驗證細節**
> - **帳戶名稱：** 必須在 3 到 20 個字元之間，不應為 null 或空。
> - **全名：** 不應為空且必須少於 50 個字元。
> - **聯絡電子郵件：** 必須是使用正規表達式的有效電子郵件格式。
> - **密碼雜湊：** 必須是 64 個字元的字串（假設使用 SHA-256 雜湊），不應為 null 或空。
> - **最近登入時間：** 如果提供的列表為 null，則初始化一個空列表以避免 NullPointerException。

像往常一樣，答案有點超出了實際問題。ChatGPT 提出了類別的完整結構，定義了私有實例變數，並提供了用於資料存取的 getter 和 setter 方法。驗證在 setter 方法中進行。我們質疑這個概念。

**提示：** Java: Using getters and setters only seems to complicate my code. What's the benefit?

在其回應中，ChatGPT 解釋說所選的方法在這裡實際上是合理的。因此我們遵循建議並實作類別，包括屬性、getter 和 setter 方法以及建構子。
