# 12.4 建立索引

正如我們在第 12.1 節中所看到的，LlamaIndex 提供了一個非常方便的函式，用於將文件轉換為可搜尋的索引：VectorStoreIndex.from_documents。這個函式僅需要文件列表作為參數。LlamaIndex 在背景中執行了一些有趣的步驟，我們將在本節中仔細探討。

首先，從 LlamaIndex 文件建立 LlamaIndex 節點。為此，使用了文字分割器（text splitter），例如 SentenceSplitter。與「普通」的文字分割器不同，這個分割器重視不將一個句子分割成多個節點。

為了讓你對使用 SentenceSplitter 將文件分割成節點的數量有個概念，我們計算了範例中 18 份文件的 PDF 頁數（當然是借助 Claude AI 工具建立的 Linux 命令），並將其與生成的節點進行比較。378 頁 PDF 變成了 466 個 LlamaIndex 節點。我們以 JSON 結構的形式為你列印了這樣一個節點的部分內容（不包含嵌入）：

```json
{
  "page_label": "5",
  "file_name": "Vielfalter-Falter_Tirol_2018_bis_2022.pdf",
  "file_path": "/data/pdfs/Vielfalter-Falter_Tirol_2018_bis_...",
  "file_type": "application/pdf",
  "file_size": 3190932,
  "creation_date": "2024-07-23",
  "last_modified_date": "2024-07-23",
  "_node_content": "{\"id_\": \"2d677c1e-da77-459d-9282...\",
     \"text\": \"Im Expert*innen-Monitoring werden inner ...",
     \"class_name\": \"TextNode\"}",
  "_node_type": "TextNode",
  "document_id": "94bdaa9f-c86c-4cf2-b38b-6d03805d265c",
  "doc_id": "94bdaa9f-c86c-4cf2-b38b-6d03805d265c",
  "ref_doc_id": "94bdaa9f-c86c-4cf2-b38b-6d03805d265c"
}  
```

你會認出一些關於文件的元資料。內容（text）被分配給 _node_content 鍵。

你可以透過呼叫 VectorStoreIndex.from_documents 時的 transformations 附加參數來控制文件到節點的轉換。然後，會傳輸一個要處理的函式列表：

```python
from llama_index.core import VectorStoreIndex
from llama_index.core.extractors import TitleExtractor
from llama_index.core.node_parser import SentenceSplitter
from llama_index.embeddings.openai import OpenAIEmbedding
index = VectorStoreIndex.from_documents(
    documents=documents,
    transformations=[
        SentenceSplitter(),
        TitleExtractor(),
        OpenAIEmbedding(),
    ],
    show_progress=True,
)  
```

在 SentenceSplitter 之後，我們使用 TitleExtractor，這是 LlamaIndex 函式庫中的另一個模組。它嘗試在 LLM 的幫助下為分割後的文字生成簡短摘要。（我們稍後會展示如何設定 LLM。）這個過程會顯著減慢索引速度，因為 466 個節點中的每一個都需要查詢 LLM，但也會在後續搜尋中帶來更好的結果。

請注意，雲端中的 LLM 通常在一定時間內可以進行的 API 呼叫次數有限制。在 Anthropic，我們使用的 Claude 3.5 Sonnet 模型的 API 限制為每分鐘 50 個請求和 40,000 個 token。僅請求限制一項，在幾分鐘內執行的 466 個請求就很容易超過。因此，我們將 TitleExtractor 與本地 Ollama 實例一起使用，那裡的限制僅由硬體決定。

轉換列表中的第三個條目涉及嵌入模型。這正是令人興奮的地方，因為模型對生成的索引有重大影響，而模型的選擇幾乎是無限的。

## 12.4.1 嵌入模型

我們使用嵌入模型將 PDF 文件中的文字段落和載入的網頁轉換為向量。然而，一個向量不代表一個詞或一個 token，而是代表整個段落或整個頁面。我們將展示這樣一個向量的範例。根據訓練資料的大小和向量的維度數，嵌入模型的大小可以從幾百 MB 到幾 GB 不等。另一方面，我們稍後將儲存的衍生向量對於我們的資料集只佔用幾 MB。

前面顯示的 OpenAIEmbedding() 呼叫會從 OpenAI 載入專有的 text-embedding-ada-002-v2 模型。要使用它，你需要一個 API 金鑰和 OpenAI 帳戶中的額度。然而，你也可以使用來自 Hugging Face 平台的開源模型：

```python
from llama_index.embeddings.huggingface import (
    HuggingFaceEmbedding
)
embed_model_hf = HuggingFaceEmbedding(
  cache_folder="/home/user/huggingface_cache",
  model_name="BAAI/bge-m3",
)  
```

為 Hugging Face 指定 cache_folder 是合理的，否則模型必須一次又一次地從網路重新載入。即使嵌入模型不像本地 LLM 那麼大，它們也很快就會累積到幾 GB。

這裡使用的 BAAI/bge-m3 已經用超過 100 種語言進行了訓練，並在自由的 MIT 授權條款下提供。向量有 1,024 個維度，模型可以處理最多 8,192 個 token 的輸入。然而，超過 8 GB 的記憶體需求不應被低估。如果你只處理英文文字，小得多的 BAAI/bge-base-en-v1.5 模型是一個好的替代方案。

**Hugging Face，AI 模型的平台**

在 https://huggingface.co 你會找到大量的模型，以及 Massive Text Embedding Benchmark 排行榜（https://huggingface.co/spaces/mteb/leaderboard），這是目前模型及其參數的排名。

如果你正在尋找適合你目的的正確模型，可以在這裡按主題或關鍵字進行篩選。你還會找到許多免費模型的訓練資料。

如果你運行本地 Ollama 伺服器，你也可以在那裡安裝嵌入模型並用它們來生成索引：

```python
from llama_index.embeddings.ollama import OllamaEmbedding
embed_model_ol = OllamaEmbedding(model_name="mxbai-embed-large")  
```

這裡的 mxbai-embed-large 使用 1,024 個維度，記憶體需求為 670 MB。在我們使用 PDF 文件和網站的測試中，該模型提供了有用的結果，雖然不如 Hugging Face 上大得多的 BAAI/bge-m3 那麼好。

但嵌入模型究竟用來做什麼？輸入資料與訓練好的模型資料進行比較，從中推導出輸入資料的向量。前面顯示的 LlamaIndex 節點的嵌入（即向量）如下所示：

```
[ 0.010941985063254, -0.007302129175513, 0.017867475748062, ... ]  
```

我們沒有列印剩餘的 765 個浮點數。這個實驗中使用的開源 BAAI/bge-base-en-v1.5 模型使用了 768 個維度。

建立索引是一個運算密集的操作。特別是如果你使用本地模型（在 Ollama 中或來自 Hugging Face），強大的圖形處理單元（GPU）會帶來顯著的差異。NVIDIA 的現代顯示卡或現代 Apple 電腦中的處理器目前提供了最好的支援。運算時間也取決於所使用的模型。更多的維度和更大的模型需要更多的運算時間，但也提供更好的結果。

對於來自雲端的嵌入模型，你不需要自己擁有強大的硬體，因為這些工作在所選供應商的資料中心中進行。然而，為了計算向量，你索引的所有資料都必須上傳到雲端。當然，我們不知道供應商在計算後對你的資料做了什麼，但我們認為它被簡單刪除的可能性極低。

除了 OpenAI 之外，當然還有其他供應商可以租用嵌入模型：Google、MistralAI、CohereAI、IBM、Oracle、Nvidia 等等——這份名單讀起來就像 IT 產業的名人錄。使用 LlamaIndex 函式庫的範例可以在 https://docs.llamaindex.ai/en/stable/examples 的 Embeddings 部分找到。

你還應該記住，嵌入模型不僅用於建立索引，還用於你提出的每個問題。正如我們將在第 12.6 節中看到的，用於建立索引的相同嵌入模型也必須用於將你的問題轉換為向量。
