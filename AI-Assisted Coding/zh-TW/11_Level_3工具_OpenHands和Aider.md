# 第 11 級 Level 3 工具：OpenHands 和 Aider

## 概述

在 AI 輔助程式設計的背景下，Level 3 工具代表了自主性和能力的重大提升。雖然 Level 1 和 Level 2 工具在您的開發環境中運作，但 Level 3 工具擁有自己獨立的環境，並且可以自主執行複雜的多步驟任務。本章介紹了兩個強大的 Level 3 工具：**OpenHands** 和 **Aider**。

## 什麼是 Level 3 工具？

Level 3 AI 程式設計工具的特點是它們能夠：

- **擁有獨立的執行環境**：它們在沙盒或容器化環境中運作，而不僅僅是與您的編輯器整合。
- **自主執行多步驟任務**：它們可以獨立規劃、執行和驗證複雜的任務序列。
- **與完整的開發工具鏈互動**：它們可以執行測試、執行終端機命令、管理檔案等等。
- **提供工作歷程的可見性**：它們記錄其行動和決策過程。

## OpenHands

OpenHands（前身為 OpenDevin）是一個開源的自主 AI 軟體工程師代理。它提供了一個完整的開發環境，AI 代理可以在其中：

- 編寫和修改程式碼
- 執行命令和腳本
- 瀏覽網路以獲取資訊
- 與終端機和檔案系統互動
- 除錯和測試程式碼

### OpenHands 的主要特性

1. **沙盒執行**：在安全的容器化環境中運作
2. **多代理架構**：支援不同的代理策略
3. **網頁式介面**：透過瀏覽器提供視覺化介面
4. **整合工具**：包含終端機、程式碼編輯器和瀏覽器工具
5. **可擴展**：可搭配各種 LLM 提供者使用

## Aider

Aider 是一個命令列 AI 程式設計工具，可直接與您的 Git 儲存庫協同運作。它專為希望由 AI 處理實際程式碼變更，同時保持完整版本控制的開發者而設計。

### Aider 的主要特性

1. **Git 整合**：自動提交 AI 生成的程式碼變更
2. **多檔案編輯**：可以同時修改多個檔案
3. **廣泛的語言支援**：適用於幾乎所有程式設計語言
4. **多種 LLM 支援**：可搭配各種語言模型使用
5. **架構感知**：理解您的程式碼庫結構
6. **互動式工作階段**：與 AI 進行對話式程式設計

## Level 3 工具的比較

| 特性 | OpenHands | Aider |
|-------|-----------|-------|
| 介面 | 網頁式 | 命令列 |
| 環境 | 沙盒容器 | 本地 Git 儲存庫 |
| 自主性 | 高度自主 | 互動式引導 |
| Git 整合 | 可用 | 核心功能 |
| 多檔案支援 | 是 | 是 |
| 測試執行 | 內建 | 透過命令可用 |

## 何時使用 Level 3 工具

Level 3 工具最適合：

- **大型重構任務**：當您需要跨多個檔案進行重大變更時
- **探索性開發**：當您想要原型化或試驗想法時
- **程式碼庫理解**：當您需要理解並導航不熟悉的程式碼時
- **自動化測試**：當您需要生成並執行測試時
- **程式碼遷移**：當您將程式碼從一個框架或版本遷移到另一個框架或版本時

## 總結

Level 3 工具代表了我們與 AI 協助程式設計方式演進的重要一步。透過提供自主執行環境和處理複雜的多步驟任務，OpenHands 和 Aider 等工具使開發者能夠專注於更高層次的設計和架構決策，而 AI 則處理實作細節。後續章節將詳細探討如何有效地使用這兩個工具。
