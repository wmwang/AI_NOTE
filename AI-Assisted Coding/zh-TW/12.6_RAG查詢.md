# 12.6 RAG 查詢

為了讓我們的問題得到有意義的答案，LlamaIndex 函式庫會依序執行以下幾個步驟：

1. 自然語言的問題被轉換為向量。為此，必須使用與建立 VectorStoreIndex 時相同的嵌入模型。
2. 系統在資料庫中搜尋相似的向量。
3. 找到的 LlamaIndex 節點內容連同問題一起被轉送到 LLM 作為上下文。
4. LLM 根據資料用自然語言組織答案。

到目前為止我們已經討論了嵌入模型，這是我們第一次接觸 LLM。但其實不完全是：在第 12.4 節中，TitleExtractor 已經使用了 LLM 來從分割後的文件建立摘要。那麼使用的是哪個 LLM，我們又可以在哪裡設定它呢？

LlamaIndex 目前預設使用 OpenAI 的 gpt-4o 作為 LLM，前提是你已在 OPENAI_API_KEY 環境變數中儲存了有效的 API 金鑰。然而，你可以非常輕鬆地更改這個行為，例如使用來自 Ollama 伺服器的本地 llama3.1：

```python
from llama_index.core.settings import Settings
from llama_index.llms.ollama import Ollama
...
Settings.llm = Ollama(model="llama3.1")  
```

或者，你也可以使用熱門的 Claude 3.5 Sonnet：

```python
from llama_index.llms.anthropic import Anthropic
...
Settings.llm = Anthropic(model="claude-3-5-sonnet-20240620")  
```

請注意設定的匯入和隨後的 Settings.llm 設定。LlamaIndex 現在會將設定的模型用於所有對 LLM 的存取。嵌入模型也是如此。對應的設定是 Settings.embed_model。

**提示：** Which butterfly species was found most frequently in Tyrol in 2023?

要提出這個問題，我們仍然需要一個 LlamaIndex query_engine，可以從 VectorStoreIndex 中衍生：

```python
query_engine = index.as_query_engine()
response = query_engine.query(
  "Which butterfly species was found "
  "most frequently in Tyrol in 2023?")
print(response)  
```

不同模型的不同答案令人著迷。前面的問題涉及奧地利昆蟲監測 2023 年年度報告第 41 頁的一個表格。該 PDF 文件包含關於科學結果的廣泛資訊，不僅限於提洛爾的調查。

以下是 bge-m3 嵌入模型和 Claude 3.5 Sonnet LLM 組合的答案：

> 大眼蝶（Maniola jurtina）是 2023 年在提洛爾最常被觀察到的蝴蝶物種。共有 430 個個體被計數，是該年被觀察到次數最多的物種。

當我們使用 OpenAI 嵌入模型時，資訊更加詳細：

> 大眼蝶（Maniola jurtina）是 2023 年在提洛爾最常被觀察到的蝴蝶物種。該物種在計數個體數最多的列表中保持首位。其後按頻率依次是小菜粉蝶（Pieris rapae）、普通藍蝶（Polyommatus icarus）和小草地蝶（Coenonympha pamphilus）。
>
> 然而，值得注意的是，小狐蝶（Aglais urticae）是分佈最廣的物種，在調查的 100 個地點中的 87 個被發現。

如果你曾經嘗試自己從 PDF 文件的表格中提取資料，你就會知道在大多數情況下，重新輸入資料會更快。PDF 描述的是頁面上某個東西的位置，而不是內容上如何關聯。這使得向量搜尋和 LLM 的組合效果更加令人驚嘆。Claude 指出小狐蝶在最多地點被發現這一點很值得注意，即使數字 87 指的是 2018 年到 2023 年的整個期間，而不是所查詢的 2023 年。

你可以從計畫網站 https://viel-falter.at/ergebnisse/veroeffentlichungen 下載 PDF 文件，以獲得對原始資料的自己印象。

## 12.6.1 原始碼

在展示了所有程式碼片段之後，我們想展示用於測試模型的完整程式。它包含用於選擇嵌入模型和 LLM 的命令列參數，以及重新生成 VectorStoreIndex 的開關和更多日誌輸出的開關。我們沒有列印許多匯入語句，因為它們對理解程式不是必須的。

請注意，在建立 VectorStoreIndex 時，args.embed 開關的值被作為集合名稱傳輸。如果未指定此開關，將使用 openai 預設值。這讓你可以將多個索引並行儲存在 Milvus 資料庫中並並行測試。

還有一點關於 OpenAI 和 Anthropic 的環境變數的說明：我們在當前目錄中建立了一個包含以下內容的 .env 檔案：

```
OPENAI_API_KEY="sk-proj-dG...."
ANTHROPIC_API_KEY='sk-ant-api03-....'  
```

Python 函式庫 dotenv（可透過 pip install dotenv 安裝）會載入這個檔案。只要你的帳戶中還有一些額度，你就可以嘗試來自 OpenAI 或 Anthropic 的各種模型。

```python
# File: vf-rag.py
load_dotenv()
parser = argparse.ArgumentParser(
    prog="vf-rag",
    description="Information about viel-falter.at project")
parser.add_argument("-v", "--verbose", action="store_true")
parser.add_argument("-d", "--debug", action="store_true")
parser.add_argument("-r", "--regenerate", action="store_true")
parser.add_argument(
    "-l", "--llm", choices=["ollama", "openai", "claude"],
    default="claude")
parser.add_argument(
    "-e", "--embed", default="openai",
    choices=["m3", "nomic", "mxbai", "base", "openai"])
args = parser.parse_args()
if args.verbose or args.debug:
    logging.basicConfig(
        stream=sys.stdout,
        level=logging.DEBUG if args.debug else logging.INFO)
    logging.getLogger().addHandler(logging.StreamHandler(
      stream=sys.stdout))
milvus_store = "http://localhost:19530"
dimensions = 1024
if args.embed == "m3":
    Settings.embed_model = HuggingFaceEmbedding(
        model_name="BAAI/bge-m3",
        cache_folder="/home/user/huggingface_cache")
elif args.embed == "base":
    Settings.embed_model = HuggingFaceEmbedding(
        model_name="BAAI/bge-base-en-v1.5",
        cache_folder="/home/user/huggingface_cache")
    dimensions = 768
elif args.embed == "nomic":
    Settings.embed_model = OllamaEmbedding(
    model_name="nomic-embed-text")
    dimensions = 512
elif args.embed == "mxbai":
    Settings.embed_model = OllamaEmbedding(
    model_name="mxbai-embed-large")
else:
    Settings.embed_model = OpenAIEmbedding()
    dimensions = 1536
if args.llm == "ollama":
    Settings.llm = Ollama(model="llama3.1")
elif args.llm == "openai":
    Settings.llm = OpenAI(model="gpt-4o-mini")
else:
    Settings.llm = Anthropic(
      model="claude-3-5-sonnet-20240620")
logging.log(level=logging.INFO, msg=args)
logging.log(level=logging.DEBUG, msg=os.environ)
if args.regenerate:
    pdf_documents = SimpleDirectoryReader("pdfs").load_data(
      show_progress=True)
    webpages = [
        "https://viel-falter.at/",
        "https://viel-falter.at/monitoring/ziele-vision/",
        "https://viel-falter.at/projektpartner/",
        "https://viel-falter.at/news/",
        "https://viel-falter.at/tagfalter-exkursion/",
    ]
    web_documents = SimpleWebPageReader(
      html_to_text=True).load_data(webpages)
    documents = pdf_documents + web_documents

    vector_store = MilvusVectorStore(
        uri=milvus_store, dim=dimensions, overwrite=True,
        collection_name=args.embed)
    storage_context = StorageContext.from_defaults(
      vector_store=vector_store)
    index = VectorStoreIndex.from_documents(
        documents=documents,
        storage_context=storage_context,
        show_progress=True)
else:
    vector_store = MilvusVectorStore(
        uri=milvus_store, dim=dimensions,
        collection_name=args.embed)
    index = VectorStoreIndex.from_vector_store(
        vector_store)
query_engine = index.as_query_engine()
response = query_engine.query(
  "Which butterfly species was found "
  "most frequently in Tyrol in 2023?")
print(response)  
```

## 12.6.2 結論

我們對於使用 LlamaIndex 函式庫即使使用預設設定也能達成如此多的成果感到印象深刻。剛才展示的 RAG 腳本的整個原始碼僅包含約 100 行程式碼，其中我們還包含了我們測試中使用的 LLM 命令列參數和索引重新建立的開關。

我們不想讓你相信可以用這麼簡單的技巧建立一個能在生產環境中成功使用的應用程式。有幾個例子表明快速拼湊的聊天機器人並不是很大的成功。LLM 的偏差等問題不容忽視。此外，應用程式還應該防範創意使用者的輸入，這些輸入試圖將 LLM 用於你所需查詢以外的目的。

然而，對於公司內部的知識資料庫來說，這可能是一個成功應用程式的起點。毫無疑問，應該持續進行測試，根據預先定義的規則檢查答案的品質。
