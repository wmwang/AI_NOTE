# 10.5 範例：從 Python 2 到 Python 3

在本節中，我們將探討一個使用 AI 輔助程式設計工具將 Python 2 程式碼遷移到 Python 3 的實際範例。此範例示範了 AI 工具如何協助處理程式碼遷移和現代化的複雜過程。

## Python 2 到 Python 3 的遷移挑戰

Python 2 到 Python 3 的遷移是一項常見但具挑戰性的任務，涉及許多語法和功能上的變更，包括：

- **print 語句**：從 `print "hello"` 變更為 `print("hello")`
- **整數除法**：從 `/` 變更為 `//` 以進行整數除法
- **字串處理**：Unicode 字串成為預設
- **例外處理語法**：從 `except Exception, e:` 變更為 `except Exception as e:`
- **range 與 xrange**：`xrange()` 已被移除，由 `range()` 取代
- **字典方法**：`.iteritems()` 變更為 `.items()`

## 使用 AI 輔助遷移

**提示：**

```
I have the following Python 2 code that I need to migrate to Python 3. 
Please help me convert it:

```python
def process_data(items):
    results = []
    for item in items:
        if isinstance(item, basestring):
            results.append(item.upper())
        elif isinstance(item, (int, long)):
            results.append(item * 2)
    print "Processed", len(results), "items"
    return results

data = {u"name": u"test", u"values": xrange(10)}
for key, value in data.iteritems():
    print key, "=>", value
```

Please identify all Python 2 specific syntax and convert them to Python 3 equivalents.
```

**AI 回應：**

以下是 Python 3 轉換後的程式碼，並附帶所有變更的說明：

```python
def process_data(items):
    results = []
    for item in items:
        if isinstance(item, str):  # basestring -> str
            results.append(item.upper())
        elif isinstance(item, int):  # (int, long) -> int
            results.append(item * 2)
    print("Processed", len(results), "items")  # print 語句 -> print 函式
    return results

data = {"name": "test", "values": range(10)}  # u"" 前綴不再需要，xrange -> range
for key, value in data.items():  # iteritems -> items
    print(key, "=>", value)  # print 語句 -> print 函式
```

### 變更說明：

1. **`basestring` → `str`**：在 Python 3 中，所有字串都是 Unicode 字串，因此 `basestring` 已被移除，直接使用 `str` 即可。

2. **`(int, long)` → `int`**：Python 3 中的 `int` 型別結合了 Python 2 的 `int` 和 `long`，因為整數的精度不再受限。

3. **`print` 語句 → `print()` 函式**：`print` 在 Python 3 中是函式，需要使用括號。

4. **`u""` 前綴**：雖然仍支援，但在 Python 3 中已不再需要，因為字串預設就是 Unicode。

5. **`xrange()` → `range()`**：Python 2 中的 `xrange()` 已被移除；Python 3 中的 `range()` 具有相同的記憶體效率行為。

6. **`.iteritems()` → `.items()`**：字典方法 `.iteritems()` 已被移除；Python 3 中的 `.items()` 會回傳一個視圖物件，行為類似。

## 進階遷移範例

**提示：**

```
Now I have a more complex Python 2 class that needs migration:

```python
class DataManager(object):
    def __init__(self):
        self._data = {}
    
    def add(self, key, value):
        if self._data.has_key(key):
            print "Warning: Key '%s' already exists" % key
        self._data[key] = value
    
    def get(self, key, default=None):
        try:
            return self._data[key]
        except KeyError, e:
            print "Key not found:", e
            return default
    
    def keys(self):
        return self._data.keys()
    
    def items(self):
        return self._data.items()
```

Please convert this to Python 3 and suggest any modern Python improvements.
```

**AI 回應：**

以下是 Python 3 轉換後的版本，並結合了現代化的改進：

```python
class DataManager:
    """管理具有鍵值儲存的資料。"""
    
    def __init__(self) -> None:
        self._data: dict[str, any] = {}
    
    def add(self, key: str, value: any) -> None:
        if key in self._data:  # has_key() -> in 運算子
            print(f"Warning: Key '{key}' already exists")  # f-string
        self._data[key] = value
    
    def get(self, key: str, default: any = None) -> any:
        try:
            return self._data[key]
        except KeyError as e:  # except..., e -> except...as e
            print(f"Key not found: {e}")  # f-string
            return default
    
    def keys(self) -> list[str]:
        return list(self._data.keys())  # 回傳 list 而非視圖
    
    def items(self) -> list[tuple[str, any]]:
        return list(self._data.items())  # 回傳 list 而非視圖
```

### 遷移的額外改進：

1. **`has_key()` → `in` 運算子**：`dict.has_key()` 方法已在 Python 3 中移除。`in` 運算子是慣用的替代方案。

2. **`except KeyError, e:` → `except KeyError as e:`**：舊的逗號語法已移除，改用 `as` 關鍵字。

3. **現代化增強功能**：
   - 移除了 `(object)` 基底類別（在 Python 3 中是隱含的）
   - 新增了型別提示（Python 3.5+）
   - 使用 f-string 取代 `%` 格式化（Python 3.6+）
   - 新增了 docstring
   - 明確地將字典視圖轉換為列表

## 遷移的最佳實踐

在使用 AI 工具進行 Python 2 到 Python 3 的遷移時：

1. **使用 `2to3` 工具**：Python 內建的 `2to3` 工具可以自動轉換大部分的程式碼。將此與 AI 輔助建議結合使用。

2. **徹底測試**：在遷移後，執行全面的測試以確保功能沒有被破壞。

3. **逐步遷移**：將遷移分解為更小的步驟，特別是對於大型程式碼庫。

4. **檢閱 AI 建議**：雖然 AI 工具擅長識別常見模式，但請務必檢閱建議的變更是否適合您的特定情況。

5. **考慮相容性函式庫**：對於需要同時支援 Python 2 和 3 的程式碼庫，請考慮使用 `six` 或 `future` 等函式庫。

## 總結

Python 2 到 Python 3 的遷移展示了 AI 輔助程式設計工具如何加速程式碼現代化。AI 工具可以快速識別特定版本的語法，建議正確的替代方案，甚至提出額外的現代化改進建議。然而，人為監督對於確保遷移程式碼的正確性和效能仍然至關重要。
