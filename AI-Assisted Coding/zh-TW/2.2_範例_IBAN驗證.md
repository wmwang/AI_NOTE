## 2.2.3 除錯

這些資訊足以讓我們撰寫程式的第一個基本版本，它只檢查 IBAN 的長度。然而，Python 突然報告我們的程式碼中有一個我們不理解的錯誤。我們向 ChatGPT 尋求建議。

**提示：** My editor says this code, which is part of a function, is wrong. Why?
```python

    print ('Country code not found')
    return False
else if len(iban) != iban_lengths[country_code]:
    print ('IBAN length is incorrect')
    return False
else:
    return True
```

AI 工具立即認出了錯誤（`else if` 應該改為 `elif`），但它提供了一頁半的答案，包含大量不相關的細節，這令人煩惱。

我們繼續修改我們的程式碼，並覺得它可以運作了——除了尚未實作的校驗和之外。此時，程式碼如下：

```python
#!/usr/bin/env python3
iban_lengths = {
    'AT': 20,  # 奧地利
    'DE': 22,  # 德國
    'CH': 21,  # 瑞士
}

# 檢查 IBAN 的長度
def check_iban_length(iban):
    country_code = iban[:2]
    if country_code not in iban_lengths:
        print('Country code not found')
        return False
    elif len(iban) != iban_lengths[country_code]:
        print('IBAN length is incorrect')
        return False
    else:
        return True

# 驗證 IBAN；稍後新增校驗和驗證
def validate_iban(iban):
    return check_iban_length(iban)

# 主程式碼從這裡開始
iban = 'AT4832000000123458642'
if validate_iban(iban):
    print('Valid IBAN')
```

## 2.2.5 移除 IBAN 中的空格

**提示：** Now I have a long string of numbers. How do I convert it to an integer?

**提示：** Is there no risk of an overflow error? The numbers are quite large.

經過一小時的工作，我們的程式碼如下：

```python
# 測試校驗和
def test_checksum(iban):
    iban = iban[4:] + iban[:4]
    digit_string = ''
    # 遍歷 IBAN 並將字母替換為數字
    for c in iban:
        if c.isdigit():
            digit_string += c
        else:
            digit_string += str(ord(c) - 55)
    # 將字串轉換為整數並測試模 97 是否為 1
    return int(digit_string) % 97 == 1
```
