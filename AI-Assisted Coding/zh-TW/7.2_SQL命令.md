## 7.2 SQL 命令

結構化查詢語言（SQL）用於從資料庫中提取資料，以及插入、變更或刪除新資料。此外，還有無數的 SQL 命令用於建立或變更新資料庫以及執行管理任務。

在本節中，我們將提供一些範例，說明 ChatGPT 如何幫助您開發正確的 SQL 命令。SQL 的語法由標準定義；因此，SQL 在所有關聯式 DBMS 中的運作方式大部分相同。

然而，幾乎所有 DBMS 都具有製造商特定的標準偏差。因此，重要的是您始終指定您使用的是哪個程式。

### 7.2.1 日期計算

您經常需要處理日期或時間。但是如何確定過去六個月的訂單或過去三小時的日誌條目？如何為一週後的會議設定提醒？時間和日期計算的函式因 DBMS 而異。

**提示：** MySQL: How do I get the date six months before now?

在 MySQL 中，您可以使用 `DATE_SUB` 函式來獲取當前日期六個月前的日期。以下是具體做法：

```sql
SELECT DATE_SUB(NOW(), INTERVAL 6 MONTH) AS 
six_months_ago; 
```

此命令將回傳恰好當前日期和時間六個月前的日期和時間。如果您只需要日期部分而不需要時間，可以使用 `CURDATE()` 代替 `NOW()`：

```sql
SELECT DATE_SUB(CURDATE(), INTERVAL 6 MONTH) AS 
six_months_ago; 
```

### 7.2.2 疑難排解

SQL 語法的特殊性有時會讓人發瘋。AI 工具在許多情況下都能成功解決這些難題。在簡單的情況下，您甚至不需要告訴 AI 工具資料庫的結構。對於跨多個表格連結的查詢，如果 ChatGPT 等工具知道表格欄位的資料類型，它們可以提供更精確的建議。

**提示：** I want to find categories with more than 10 titles in my book library. However, my query fails (syntax error). Why is this?

```sql
SELECT c.name AS categoryName, COUNT(*) AS cnt 
FROM books b JOIN categories c ON c.id = b.catId 
GROUP BY c.id 
WHERE cnt >= 10 
```

您的查詢問題在於 `WHERE` 子句的位置。在 SQL 中，`WHERE` 子句在 `GROUP BY` 子句之前使用。如果您想篩選 `GROUP BY` 操作的結果，應該使用 `HAVING` 子句。以下是修正查詢的方法：

```sql
SELECT c.name AS categoryName, COUNT(*) AS cnt
FROM books b JOIN categories c ON c.id = b.catId 
GROUP BY c.id 
HAVING cnt >= 10 
```

我們在 VS Code 中使用 GitHub Copilot 執行了第二個範例。AI 工具可以存取資料庫結構檔案，甚至在回覆中說明它使用了此資訊。

**提示：** In my accounting database, I want to create a table of all customers with the total number of invoices. However, new customers who haven't received an invoice yet are missing from the result. What's wrong with my query?

```sql
SELECT company, COUNT(*) 
FROM customers c JOIN invoices i on i.customerId = 
c.id 
GROUP BY c.id 
```

您的查詢問題在於您使用的是 `INNER JOIN`，它只回傳兩個表格中有匹配的行。如果客戶還沒有發票，他們就不會出現在結果集中。

要包含尚未收到發票的客戶，您應該使用 `LEFT JOIN`。這種類型的連結會回傳左表格（customers）的所有行，以及右表格（invoices）中匹配的行。如果沒有匹配，右側的結果為 `NULL`。

以下是修改查詢的方法：

```sql
SELECT c.company, COUNT(i.invoiceId) 
FROM customers c 
LEFT JOIN invoices i ON i.customerId = c.id 
GROUP BY c.id, c.company; 
```

此查詢將回傳所有客戶，對於沒有發票的客戶，將顯示計數為 0。

GitHub Copilot 正確識別了錯誤的原因，並且基本上提出了一個好的解決方案。令人懊惱的是，它引入了一個本可以避免的新錯誤：在 `COUNT(i.invoiceId)` 中，AI 工具假設 invoices 表格的 ID 欄位名為 `invoiceId`。這看起來可能很明顯，但在這個特定情況下，該欄位簡單地命名為 `id`。此資訊可以從結構檔案中清楚得知。

### 7.2.3 WordPress 圖片搜尋

最近，我們想要在 WordPress 資料庫中為圖片新增 alt 屬性作為額外的圖片描述。我們知道 WordPress 使用 MySQL 或 MariaDB 作為 DBMS，但我們之前從未研究過此資料庫的結構。ChatGPT 派上了用場。

**提示：** WordPress/MySQL: How can I get the newest 10 uploaded images with URL, description, and alt text from the WordPress database?

要檢索最新的 10 張上傳圖片，您需要在 WordPress MySQL 資料庫上執行 SQL 查詢。WordPress 將上傳圖片（以及所有媒體庫項目）的資訊儲存在 `wp_posts` 表格中，並將相關的元資料儲存在 `wp_postmeta` 表格中。圖片的 URL、描述和 alt 文字都作為元資料儲存。

以下查詢是如何檢索此資訊的範例。此查詢假設您使用預設的 WordPress 表格前綴（`wp_`）。如果您有不同的表格前綴，則需要在查詢中將 `wp_` 替換為您的自訂前綴：

```sql
SELECT p.ID,
       p.guid AS 'URL',
       p.post_content AS 'Description',
       pm.meta_value AS 'Alt Text'
FROM  wp_posts p
LEFT JOIN wp_postmeta pm ON p.ID = pm.post_id 
       AND pm.meta_key = '_wp_attachment_image_alt'
WHERE p.post_type = 'attachment'
  AND p.post_mime_type LIKE 'image/%'
ORDER BY p.post_date DESC
LIMIT 10; 
```

### 7.2.4 複雜查詢與子查詢或自我連結

以下範例的出發點是來自雲端服務提供商的資料庫。每月訂閱費由 `subscriptionPrice` 欄位決定。要確定預期的每月總收入，必須將每個客戶最後一筆訂單的 `basePrice` 加總。從 SQL 的角度來看，困難之處在於並非 `orders` 表格中的所有資料記錄都會被考慮，而只考慮每個客戶的最後一筆訂單。

要讓 ChatGPT 能夠建立合適的 SELECT 查詢，您必須告知 AI 工具資料庫的結構。最簡單的方法是指定建立表格所需的 CREATE TABLE 命令。（在 MySQL 和 MariaDB 中，您可以使用 `SHOW CREATE TABLE` 確定這些命令。另一個選擇是建立結構定義的備份。）

在此範例中，我們只指定了與範例相關的欄位，並刪除了所有其他欄位。

**提示：** MySQL database: I have a database with two tables customers and orders:

```sql
CREATE TABLE customers (
  id                INT NOT NULL AUTO_INCREMENT,
  company           VARCHAR(100) NOT NULL,
  state             VARCHAR(100) NOT NULL,
                    ON UPDATE current_timestamp(),
  PRIMARY KEY (id),
)
CREATE TABLE orders (
  id                INT NOT NULL AUTO_INCREMENT,
  customerId        INT NOT NULL,
  orderType         VARCHAR(20) NOT NULL,
  orderDate         DATE NOT NULL DEFAULT curdate(),
  subscriptionPrice DOUBLE NOT NULL DEFAULT 0,
  PRIMARY KEY (id),
  CONSTRAINT orders_ibfk_1 FOREIGN KEY (customerId) 
    REFERENCES customers (id)
) 
```

I need to calculate a sum over subscriptionPrice for all customers with customers.state = 'ok'. The sum should only include the newest order per customer with orders.state = 'subscription'. Can you help me with the SELECT command?

當然！要計算所有 `state = 'ok'` 的客戶的 `subscriptionPrice` 總和，且只包含每個客戶 `orderType = 'subscription'` 的最新訂單，您可以使用子查詢來獲取每個客戶的最新訂單，然後將其與 `customers` 表格連結。以下是撰寫 SQL 查詢的方法：

```sql
SELECT SUM(o.subscriptionPrice) AS total
FROM customers c
JOIN (
    SELECT customerId, MAX(orderDate) AS 
latestOrderDate
    FROM orders
    WHERE orderType = 'subscription'
    GROUP BY customerId
) latest_orders ON c.id = latest_orders.customerId
JOIN orders o ON latest_orders.customerId = 
o.customerId 
             AND latest_orders.latestOrderDate = 
o.orderDate
WHERE c.state = 'ok'; 
```

此查詢的運作方式如下：

- 子查詢 `latest_orders` 檢索 `orderType` 為 `'subscription'` 的每個客戶的 `customerId` 和最大 `orderDate`。
- 此子查詢結果與 `customers` 表格連結，以獲取 `state = 'ok'` 的客戶。
- 再與 `orders` 表格進行另一次連結，以獲取每個客戶最新訂單的詳細資訊。
- 最後，使用 `SUM` 函式計算這些訂單的 `subscriptionPrice` 總和。

如果一個客戶在同一天有多筆訂單，此查詢將包含所有這些訂單。如果您只想每個客戶一筆訂單，即使有並列情況，您可能需要更複雜的查詢來處理平局邏輯。

這個結果非常棒！ChatGPT 不僅理解了我們的意圖，AI 工具還提供了相當完美的答案，並立即指出了唯一的限制：如果客戶在一天內下了多筆訂單，它們都會被計入。（在此範例中，由於從 `AUTO_INCREMENT` 可知 `id` 欄位包含遞增值，可以使用此欄位代替 `lastOrderDate` 來連結到最後一筆訂單。）

ChatGPT 在其建議的解決方案中使用了子查詢。這提供了相對可讀的 SQL 程式碼，但有時執行速度較慢。即使在此範例中不預期會有問題，我們想知道 ChatGPT 是否也能提供不含子查詢的解決方案。

### 7.2.5 格式化 SQL 命令

在我們的程式設計實踐中，我們通常先互動式地嘗試新的 SQL 命令，然後再將最終命令轉移到程式碼檔案中。在這些實驗中，我們不會費心以可讀的方式格式化 SQL 命令：我們用小寫字母撰寫整個命令，不使用任何換行等等。

如果一切按預期運作，正確格式化程式碼對我們來說很重要——使用大寫的 SQL 關鍵字、合理的換行等等。這在 VS Code 中運作得很好，您標記 SQL 程式碼，然後透過 Copilot 聊天請求可讀的格式化：

**提示：** Please capitalize SQL keywords and insert line breaks.

或者，您可以將 SQL 程式碼複製到聊天視窗。我們在這裡這樣做是為了在書中更好地說明概念：

**提示：** Please reformat the following SQL command and capitalize all keywords.

```sql
create table log (id integer primary key, url text, 
ip_address text, request_time datetime); 
```

```sql
CREATE TABLE log (
    id INTEGER PRIMARY KEY,
    url TEXT,
    ip_address TEXT,
    request_time DATETIME
); 
```

如果在欄位名稱和資料類型之間加上一些空格，讓資料類型在同一欄中對齊，程式碼會更容易閱讀。

**提示：** Great. Could you also add spaces between the column names and their types to align the types vertically?

```sql
CREATE TABLE log (
    id            INTEGER PRIMARY KEY,
    url           TEXT,
    ip_address    TEXT,
    request_time  DATETIME
); 
```

當然，這只有在較長的命令中才值得，否則提示輸入所花費的時間比手動重新格式化程式碼還要多。
