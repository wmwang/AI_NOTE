# 6.3 函式與類別文件

## 何謂函式與類別文件？

函式與類別文件是程式碼中對函式（Function）和類別（Class）的正式說明文件。它們通常位於函式或類別定義的上方，以結構化的格式描述其用途、參數、回傳值、例外行為等資訊。

良好的函式與類別文件能夠：

- 讓開發者在不閱讀實作程式碼的情況下理解如何使用
- 作為 API 文件的基礎來源
- 協助 IDE 提供智能提示（IntelliSense）
- 降低錯誤使用的風險

## 各語言的文件格式

### Python - Docstring

Python 使用 docstring 作為文件標準，支援多種格式：

**Google 風格：**
```python
def calculate_discount(price: float, discount_rate: float, min_price: float = 0) -> float:
    """計算折扣後的價格。

    根據給定的折扣率計算最終價格，確保不低於最低價格限制。

    Args:
        price: 原始價格，必須為正數。
        discount_rate: 折扣率，範圍為 0.0 到 1.0。
        min_price: 最低價格限制，預設為 0。

    Returns:
        折扣後的價格。

    Raises:
        ValueError: 當 price 為負數或 discount_rate 不在有效範圍內時。

    Examples:
        >>> calculate_discount(100.0, 0.2)
        80.0
        >>> calculate_discount(100.0, 0.9, min_price=20.0)
        20.0
    """
    if price < 0:
        raise ValueError("價格必須為正數")
    if not 0 <= discount_rate <= 1:
        raise ValueError("折扣率必須在 0.0 到 1.0 之間")

    discounted = price * (1 - discount_rate)
    return max(discounted, min_price)
```

## 類別文件的最佳實踐

### 類別層級文件

類別文件應說明：

1. **類別的職責**：這個類別做什麼？
2. **使用情境**：什麼時候應該使用這個類別？
3. **執行緒安全性**：是否為執行緒安全？
4. **重要注意事項**：使用時需要特別注意的事項

```python
class DataProcessor:
    """資料處理器，負責清洗和轉換原始資料。

    此類別提供一系列的資料處理方法，包括資料清洗、格式轉換
    和統計分析。所有方法都是無狀態的（stateless），可以安全
    地在多執行緒環境中使用。

    注意：輸入資料必須是 UTF-8 編碼的字串或可迭代的數值列表。

    Attributes:
        encoding: 資料的編碼格式，預設為 'utf-8'。
        strict_mode: 是否在遇到無效資料時拋出例外，預設為 True。

    Examples:
        >>> processor = DataProcessor()
        >>> cleaned = processor.clean(["hello", " world ", None])
        >>> print(cleaned)
        ['hello', 'world']
    """

    def __init__(self, encoding='utf-8', strict_mode=True):
        self.encoding = encoding
        self.strict_mode = strict_mode
```

## 文件品質檢查清單

使用以下清單確保函式與類別文件的品質：

- [ ] **用途描述**：是否清楚說明了函式/類別的用途？
- [ ] **參數說明**：所有參數是否都有說明，包含型態和有效範圍？
- [ ] **回傳值**：回傳值的型態和含義是否清楚？
- [ ] **例外說明**：所有可能拋出的例外是否都有記錄？
- [ ] **使用範例**：是否提供了至少一個使用範例？
- [ ] **邊界條件**：是否說明了特殊情況下的行為？
- [ ] **格式一致**：文件格式是否與團隊規範一致？
- [ ] **準確性**：文件描述是否與程式碼實作一致？

## 總結

函式與類別文件是程式碼品質的基石。透過 AI 輔助工具，可以大幅提升文件撰寫的效率與品質。關鍵是：

1. **保持一致性**：使用統一的文件格式
2. **確保完整性**：涵蓋所有必要的資訊
3. **善用範例**：範例是最好的文件
4. **定期審查**：確保文件與程式碼保持同步
5. **利用 AI 加速**：讓 AI 處理繁瑣的文件撰寫工作
