### 1.1.2 Python：搜尋和替換日期

在此範例中，需要將字串中的所有日期從美國格式（12/31/2024）轉換為 ISO 格式（2024-12-31）。使用的程式語言是 Python。

程式碼使用了 `re` 模組，該模組用於在 Python 中處理正規表示式。ChatGPT 將程式碼封裝為函式，這簡化了後續使用。`convert_us_to_iso_date` 反過來包含一個名為 `replace` 的子函式，它作為參數傳遞給 `re.sub`，並將每個匹配項替換為新格式的日期。

**提示：** Python：我想將字串中所有的美國日期替換為 ISO 日期。你能幫忙嗎？

當然可以！要使用 Python 將字串中所有美國日期格式（通常為 MM/DD/YYYY）替換為 ISO 日期格式（YYYY-MM-DD），您可以使用 `re` 模組來尋找日期，然後重新排列格式。以下是實現此功能的 Python 函式：

```python
import re

def convert_us_to_iso_dates(text):
    # 用於匹配 MM/DD/YYYY 格式日期的模式
    date_pattern = r'\b(\d{1,2})/(\d{1,2})/(\d{4})\b'
    
    # 替換每個匹配項的函式
    def replace(match):
        month, day, year = match.groups()
        return f'{year}-{month.zfill(2)}-{day.zfill(2)}'
    
    # 替換文字中所有匹配項
    return re.sub(date_pattern, replace, text)

# 範例使用
input_text = "Today's date is 05/09/2024 and the event was on 12/25/2023."
converted_text = convert_us_to_iso_dates(input_text)
print(converted_text)
```

此腳本：
- 定義一個正規表示式模式來匹配 MM/DD/YYYY 格式的日期。
- 使用替換函式將日期組件重新排列為 ISO 格式 YYYY-MM-DD。
- 對提供的文字中的所有出現項應用此轉換。

此處呈現的程式碼以這種形式完美運作。如果您不熟悉正規表示式，可以在後續問題中要求解釋模式的結構：

**提示：** 你能解釋一下正規表示式模式嗎？

當然可以！以下是模式 `r'\b(\d{1,2})/(\d{1,2})/(\d{4})\b'` 的詳細解析：

- `\b`：這是詞邊界標記。它確保模式只匹配完整的詞，這有助於防止部分匹配更大的非日期序列。
- `(\d{1,2})`：這部分模式捕獲一位或兩位數字。`\d` 表示一個數字（等同於 `[0-9]`），`{1,2}` 指定數字可以出現一次或兩次。此模式用於日期中的月份和日期。
- `/`：這是字面的正斜線字元。它在美國日期格式中分隔月和日，以及日和年。
- `(\d{4})`：這捕獲剛好四位數字，代表年份。這裡 `\d` 再次表示一個數字，`{4}` 指定必須剛好有四位數字。
- `\b`：另一個詞邊界標記，確保日期是一個獨立的實體，而不是更大數字或字元序列的一部分。

因此，整個模式匹配以詞邊界（如空格或標點符號）開始和結束的字串，後接月份的一位或兩位數字、一個斜線、日期的一位或兩位數字、另一個斜線，最後是年份的四位數字。

在本書中，我們通常不會複製此類解釋，或者如果 ChatGPT 的輸出過於冗長，我們會加以簡略，因為我們主要關注的是如何最好地構建提示。

### 1.1.3 SQL：分析資料表

以下範例的起點是 MySQL 中的一個資料表，包含不同國家的增值稅（VAT）率。由於稅率可能隨時間變化，`validFrom` 欄位指示該值從哪個日期開始生效。回傳某個國家在特定時間有效稅率的 SELECT 查詢應該是什麼樣子？

為了讓 ChatGPT 給出正確答案，您必須提供足夠的上下文資訊。特別是，您使用的資料庫管理系統（DBMS）（在本例中為 MySQL）以及資料表的確切結構非常重要。

**提示：** MySQL：我有一個用於儲存不同國家增值稅率的資料表。這是資料表定義：

```sql
CREATE TABLE vat (
  id        INT NOT NULL AUTO_INCREMENT,
  country   CHAR(2) NOT NULL,
  rate      DOUBLE NOT NULL,
  validFrom DATE NOT NULL,
  PRIMARY KEY (id)
)
```

我想獲取某個國家（例如 'de'）在特定時間（'2024-04-15'）的增值稅率。查詢應該是什麼樣子？

要從您的資料表中檢索特定國家在特定時間的增值稅率，您需要編寫一個 SQL 查詢，檢查在您感興趣的日期之前或當天設定的最新稅率。這涉及根據國家代碼和 `validFrom` 日期選擇正確的條目。以下是查找德國（'DE'）截至 2024 年 4 月 15 日增值稅率的 SQL 查詢：

```sql
SELECT rate 
FROM vat 
WHERE country = 'DE'
  AND validFrom <= '2024-04-15'
ORDER BY validFrom DESC
LIMIT 1;
```

ChatGPT 提供了正確的答案。然後程式詳細解釋了 SQL 指令的結構（此處未印出）。如果您不是每天都使用 SQL，這對理解程式碼是很大的幫助。

請注意，對於此類問題，您必須明確說明您使用的資料庫系統。`LIMIT 1` 是 SQL 語法的 MySQL 特定擴展。針對 SQL Server 的相同問題會提供使用 `TOP` 而不是 `LIMIT` 的不同答案。

**提示：** 這在 SQL Server 上也適用嗎？

要使其完全相容 SQL Server，唯一需要的變更是將 `LIMIT 1` 替換為 `TOP (1)`，這是 SQL Server 用於限制回傳行數的語法。以下是為 SQL Server 修改後的查詢：

```sql
SELECT TOP (1) rate 
FROM vat 
WHERE country = 'DE'
  AND validFrom <= '2024-04-15'
ORDER BY validFrom DESC;
```

這向我們展示了指定問題上下文的重要性。

### 1.1.4 PHP：Bearer Token 驗證

假設您熟悉 PHP 和 REST 應用程式設計介面（API）的概念，但不知道如何在 PHP 程式中評估 bearer token。沒問題，ChatGPT 可以幫助您：

**提示：** PHP/REST：如何從 GET 請求中提取 bearer token？

在 PHP 中，處理使用 bearer token 進行驗證的 RESTful API 請求時，您通常從傳入的 GET 請求的 `Authorization` 標頭中提取 token。以下是如何執行此操作的逐步指南：

1. **存取授權標頭**：PHP 透過 `$_SERVER` 超全域陣列提供對請求標頭的存取。可以使用此陣列中的 `HTTP_AUTHORIZATION` 鍵來存取 Authorization 標頭。
2. **提取 bearer token**：Authorization 標頭的值通常以單字「Bearer」開頭，後接一個空格，然後是 token 本身。您需要解析此字串以提取 token。

以下是一個示範如何從 GET 請求中提取 bearer token 的程式碼片段：

```php
<?php
// 從 Authorization 標頭獲取 bearer token 的函式
function getBearerToken() {
    if (isset($_SERVER['HTTP_AUTHORIZATION'])) {
        // 提取標頭值
        $header = $_SERVER['HTTP_AUTHORIZATION'];
        // 檢查標頭是否以 "Bearer" 開頭
        if (preg_match('/Bearer\s(\S+)/', $header, $matches)) {
            // 回傳 token 部分
            return $matches[1];
        }
    }
    return null; // 如果找不到 token 則回傳 null
}
?>
```

**處理邊界情況：**
- **標頭不存在**：如果 Authorization 標頭不存在，您的函式應優雅地處理此情況，可能回傳 `null` 或適當的錯誤訊息。
- **格式錯誤的授權標頭**：如果標頭存在但未正確包含 bearer token（例如，缺少「Bearer」前綴），您也應處理此情況，可能記錄問題或回傳錯誤。

**請始終確保安全地處理 token：**
- 必要時驗證 token（例如，檢查其格式、有效性和真實性）。
- 使用 HTTPS 來保護傳輸中的 token。
- 限制在日誌或錯誤訊息中暴露敏感的 token 資訊。

這種方法將幫助您在基於 PHP 的 RESTful API 環境中安全地提取和處理 bearer token。

### 1.1.5 將 Bash 腳本移植到 PowerShell

ChatGPT 不僅能產生新的程式碼。它還可以搜尋給定程式碼中的錯誤；將程式碼從一種程式語言轉換為另一種；為給定函式開發單元測試；以及重構、註解和解釋現有程式碼。

最後一個入門範例是關於將一個簡單的 Bash 腳本移植到 PowerShell。原始腳本監控目錄中的 Markdown 檔案。當檔案中儲存變更且 Markdown 檔案比對應的 PDF 檔案更新時，會使用 `pandoc` 程式重新建立 PDF 文件。這個自動化建構過程現在要在 Windows 電腦上使用 PowerShell 實作。這對 ChatGPT 來說沒問題！

**提示：** 你能提供以下 Bash 腳本的 PowerShell 等效版本嗎？

```bash
#!/bin/bash
while true; do
  for mdfile in *.md; do
    pdffile=${mdfile%.md}.pdf
    if [ $mdfile -nt $pdffile]; then
      echo $mdfile
      pandoc ... $mdfile -o $pdffile
    fi
  done
  sleep 1
done
```

當然可以！以下是給定 Bash 腳本的 PowerShell 等效版本。此腳本持續循環，檢查當前目錄中的 .md 檔案。如果 .md 檔案比對應的 .pdf 檔案更新，它會使用 Pandoc 將 Markdown 檔案轉換為 PDF。它在每次迭代後等待 1 秒。

```powershell
while ($true) {
    Get-ChildItem -Filter *.md | ForEach-Object {
        $mdFile = $_
        $pdfFile = "$($mdFile.BaseName).pdf"
        if (-Not (Test-Path $pdfFile) -or `
            (Get-Item $mdFile).LastWriteTime -gt `
            (Get-Item $pdfFile).LastWriteTime)
        {
            Write-Output $mdFile
            pandoc $mdFile -o $pdfFile
        }
    }
    Start-Sleep -Seconds 1
}
```

請確保根據您的轉換需求自訂任何 Pandoc 特定的參數。

ChatGPT「理解」了原始腳本的任務，即使它不包含任何註解。新程式碼不僅使用了 PowerShell cmdlet，還考慮到了 PowerShell 在比較兩個檔案的修改日期之前必須確保兩個檔案都確實存在這一事實。太棒了！

ChatGPT 提供的答案包含了新腳本的逐步解釋，由於篇幅原因我們這裡沒有複製。我們在此類翻譯任務中也有過負面經驗，無論程式語言為何：ChatGPT 有時會虛構出不存在的命令、cmdlet 或函式。
