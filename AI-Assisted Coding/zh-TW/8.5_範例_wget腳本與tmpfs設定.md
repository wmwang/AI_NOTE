# 8.5 範例：「wget」腳本加上「tmpfs」設定

在本節中，我們呈現一個來自實際工作的較長範例，結合了腳本程式設計與系統管理。

Linux 伺服器上使用的某個工具應該將 HTML 檔案轉換為 PDF 文件。然而，該工具相當舊的版本與現代 HTTPS 加密協定不相容。由於各種原因，升級到新版本是不可能的。

這產生了以下想法：應該可以使用 wget 將 HTML 檔案及其引用的所有額外檔案（階層樣式表 [CSS]、圖片等）下載到一個臨時目錄。然後可以在沒有網路問題的情況下將本機檔案轉換為 PDF 格式。

> **wget**
> wget 是 Linux 上常用的命令，用於透過 FTP 或 HTTP 下載檔案。wget 也有無數由選項控制的特殊功能。其中，該命令可以下載整個網站。它會調整 HTML 頁面中的連結，以便鏡像的網站可以在本機網頁瀏覽器中檢視。使用 wget 的訣竅在於傳遞正確的選項組合。

## 8.5.1 開發「wget」腳本

因此，目標是開發一個 Bash 腳本，該腳本接受所有要傳遞給工具的參數，建立一個臨時目錄，下載作為第一個參數傳入的 HTML 檔案，呼叫 HTML 轉 PDF 轉換器，最後清理所有內容。該腳本是在 VS Code 中開發的，透過編輯器中的 SSH 連線到伺服器。

**提示：** 我想編寫一個小型的 Bash 腳本，我會傳入一個 URL（第一個參數）和一些選項。該腳本建立一個臨時目錄，下載該 URL 及所有引用的檔案（wget relink），呼叫另一個命令來處理這些本機檔案，最後刪除臨時目錄。你能幫忙嗎？（偏好簡短、簡潔的程式碼。）

當然可以！以下是一個簡潔的 Bash 腳本來實現您描述的功能：

```bash
#!/bin/bash
URL=$1
shift   # shift the arguments to remove the URL

# Create a temporary directory
TMPDIR=$(mktemp -d)

# Ensure the temporary directory is deleted on script exit
trap 'rm -rf "$TMPDIR"' EXIT

# Download the URL and all referenced files into the temporary directory
wget --mirror --convert-links --adjust-extension \
     --page-requisites --no-parent -P "$TMPDIR" "$URL"

# Call another command to process these local files
your_command "$TMPDIR" "$@"
```

**說明**

- **Trap**：確保腳本退出時刪除臨時目錄，無論成功或失敗。
- **wget 命令**：使用 wget 將 URL 及所有引用的檔案下載到臨時目錄。使用的選項包括：
  - `--mirror`：啟用適合鏡像的選項。
  - `--convert-links`：轉換連結使其適合離線檢視。
  - `--adjust-extension`：為檔案添加適當的副檔名。
  - `--page-requisites`：下載顯示頁面所需的所有檔案。
  - `--no-parent`：防止 wget 追蹤給定 URL 之外的連結。
  - `-P "$TMPDIR"`：指定儲存檔案的目錄。
- **處理命令**：將 `your_command` 替換為您要在下載的檔案上執行的實際命令。腳本會將臨時目錄和任何額外選項傳遞給此命令。

腳本的第一個版本看起來已經相當不錯了。我們縮短了程式碼的說明，以保持範例的清晰。為了測試，我們幾乎原封不動地採用了這個建議，只將 `your_command` 替換為 HTML PDF 轉換器的絕對路徑。

然而，在測試過程中，我們遇到了一些問題，其中一些我們自己修復了，另一些則借助 AI 支援。以下提示記錄了這個迭代過程。我們沒有重現相當冗長的回答。

**提示：** 我需要將 `$URL` 傳遞給 `your_command`。然而，`$URL` 需要被修改以指向 wget 產生的本機起點。

**提示：** `$URL` 可以包含各種特殊字元。我能讓 wget 使用 `index.html` 作為本機檔案樹的起點嗎？

ChatGPT 找不到合適的選項（據我們所知，沒有這樣的選項），並建議使用 `find` 在臨時目錄中搜尋 `*.html` 檔案，然後使用第一個找到的檔案。然而，這在我們看來容易出錯：

**提示：** 直接取第一個結果安全嗎？也許目錄中還有其他 `*.html` 檔案。從 `$URL` 中提取基本名稱並將其作為 `find` 的搜尋模式是否有意義？

ChatGPT 現在提供了符合我們需求的程式碼。然而，事實證明 wget 對 HTML 檔案的修改過於廣泛。`wget --mirror` 不僅下載顯示 HTML 頁面所需的檔案，還會處理 HTML 頁面中包含的所有連結。因此，PDF 文件中隨後轉發的外部連結無法如預期運作。我們嘗試再次解釋我們實際想要什麼：

**提示：** 我希望 wget 在本機下載正確顯示文件所需的所有檔案（即 CSS + 圖片），並更改文件內的這些連結（僅限一份文件）。但我不希望 wget 更改其他所有內容的連結。文件中未直接顯示的外部檔案連結應保持不變。你能幫忙嗎？

該提示顯示 ChatGPT 最初建議的 `--mirror` 選項是不合適的。爭論我們一開始是否沒有精確描述任務，還是責任在於 ChatGPT，是沒有意義的。無論如何，需要一個相當長的互動過程，直到與 ChatGPT 合作產生的腳本才能如預期運作。最終的腳本如下所示：

```bash
#!/bin/bash
URL=$1
shift   # shift the arguments to remove the URL

TMP_DIR=$(mktemp -d /tmp/html2pdf-tmp.XXXXXXXX)
trap 'rm -rf "$TMPDIR"' EXIT

cd "$TMP_DIR"
wget --page-requisites --convert-links --no-parent "$URL"

# if $URL contains a long query string, wget shortens the file
# name; therefore, find the newest file in the result list
# with the same base name; this should be reasonably safe
URL_NO_QUERY=${URL%%\?*}
BASE_NAME=$(basename "$URL_NO_QUERY")
MAIN_FILE=$(find . -type f -name "${BASE_NAME}*" -print0 | \
              xargs -0 ls -t | head -n 1)

# run html2pdf
/usr/local/bin/html2pdf "$MAIN_FILE" "$@"
```

## 8.5.2 使用「tmpfs」進行 Linux 伺服器設定

該腳本現在在 Linux 伺服器上運行得非常穩定。為了提高效率，我們設定了伺服器，使 `/tmp` 目錄作為 tmpfs 檔案系統映射到工作記憶體中。想知道怎麼做嗎？問 AI 工具吧！

**提示：** 在 Ubuntu 伺服器上，我想將 `/tmp` 的內容儲存在記憶體中，以加快 I/O 速度。`/tmp` 不得使用超過 4 GB 的 RAM。請展示一個與 systemd 相容的設定。

若要使用 tmpfs 在記憶體中為 `/tmp` 建立臨時檔案系統並將其限制為 4 GB，您可以設定一個 systemd unit 來實現。以下是設定方式：

首先，建立一個新的 systemd unit 檔案，將 `/tmp` 掛載為 tmpfs，大小限制為 4 GB。

```
sudo nano /etc/systemd/system/tmp.mount
```

ChatGPT 還解釋了 `tmp.mount` 檔案的結構（12 行程式碼）及其透過 `systemctl daemon-reload` 和 `systemctl start tmp.mount` 的啟用方式。

基本上，這是一個好的解決方案，但可以更好。在現代 Linux 發行版中，`/lib/systemd/system/tmp.mount` 已經存在。只需對現有設定進行最少的修改就足夠了。只有當您將 AI 工具引導到正確的方向時，它才會描述這個程序：

**提示：** 我的 systemd 發行版已經包含一個預先配置的 `tmp.mount` 檔案。除了磁碟大小之外，它是完美的。我不能直接更改設定以使用最多 n GB 的 RAM 嗎？

是的，如果您的系統已經包含一個預先配置的 `tmp.mount` unit 檔案，您可以直接修改現有檔案將最大大小設為 4 GB。以下是步驟：

開啟 `tmp.mount` 檔案進行編輯：

```
sudo systemctl edit tmp.mount
```

在覆寫檔案中，加入以下內容以設定 tmpfs 掛載的大小限制：

```ini
[Mount]
Options=mode=1777,size=4G
```

重新載入 systemd daemon 以套用變更：

```
sudo systemctl daemon-reload
```

重新啟動 `tmp.mount` unit 以套用新的大小限制：

```
sudo systemctl restart tmp.mount
```
