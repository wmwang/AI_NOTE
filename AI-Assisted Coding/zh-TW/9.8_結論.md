# 9.8 結論

使用 LLM 在很大程度上增加了您的靈活性：您可以選擇由 OpenAI 或 Anthropic 等提供商提供的商業模型。根據合約模式，這些提供商保證您的資料受到保護，免受未經授權的存取。

如果您不信任這項保證，或者想要實驗其他語言模型，您可以在您的 PC 上、公司中央伺服器上，或是在具有 GPU 支援的雲端實例上本機執行免費的語言模型。如果公司筆記型電腦的運算能力不足以執行該 LLM 版本，後兩種方案就特別有利。無論語言模型在哪裡執行，您都可以將其作為聊天機器人使用，透過外掛作為程式碼助理，或透過 API 在您自己的應用程式中使用。

在開源領域，您可以從各種模型中進行選擇。然而，根據架構和大小的不同，它們的實作非常耗費資源。較小的版本，例如 Llama 3.1-8B，可以在配備獨立顯示卡的較高效能桌面系統上執行。然而，對於較大的模型，例如 Llama 3.1-70B 甚至 405B 變體，您需要非常強大的硬體。執行如此龐大的模型對個人或小型公司通常不划算，因為基礎設施的成本過於昂貴。在這種情況下，您需要回到其中一個商業提供商。

在我們的測試中，我們發現在私人筆記型電腦或 PC 上本機執行的 LLM 無法與商業雲端服務匹敵。這對於程式碼編寫應用尤其明顯：token 速度太低，回答品質也明顯較差。我們預期這種情況在未來會有所改變：首先，未來的電腦將提供更好的硬體條件來執行 LLM；其次，目前正進行密集的研究，以在維持相同品質的情況下減少 LLM 的大小。
