# 11.3 Aider

Aider 是一個命令列 AI 程式設計助手，可直接與您的 Git 儲存庫協同運作。它旨在成為一個 AI 結對程式設計夥伴，可以讀取、理解和修改您現有的程式碼庫。

## 什麼是 Aider？

Aider 讓您可以使用 AI 直接從終端機進行結對程式設計。它與您的 Git 儲存庫無縫整合，自動提交變更，並且可以在互動式工作階段中修改多個檔案。

### 主要特徵

- **Git 原生**：自動提交 AI 變更，並提供有意義的提交訊息
- **多檔案感知**：理解跨多個檔案的關係和相依性
- **多種 LLM 支援**：可搭配 OpenAI、Anthropic、Google 和其他 LLM 提供者使用
- **語言無關**：適用於幾乎所有程式設計語言
- **地圖功能**：建立程式碼庫的地圖以理解其結構
- **語音支援**：可用語音命令進行互動
- **圖像支援**：可以處理程式碼的圖像和螢幕截圖

## 安裝

### 使用 pip 安裝

```bash
pip install aider-chat
```

### 使用 pipx 安裝（推薦用於隔離環境）

```bash
pipx install aider-chat
```

### 其他安裝方法

```bash
# 使用 Homebrew (macOS)
brew install aider

# 使用 Docker
docker pull paulgauthier/aider
```

## 設定

### API 金鑰

設定您偏好的 LLM 提供者的 API 金鑰：

```bash
# 用於 OpenAI
export OPENAI_API_KEY="your-openai-key"

# 用於 Anthropic（推薦）
export ANTHROPIC_API_KEY="your-anthropic-key"

# 用於其他提供者
export OPENROUTER_API_KEY="your-openrouter-key"
```

### 儲存庫設定

若要在您的儲存庫中使用 Aider：

1. 導航至您的 Git 儲存庫
2. 初始化 Aider（如果需要，這會建立一個 `.aider.conf.yml` 檔案）
3. 啟動互動式工作階段

## Aider 如何運作

### 程式碼庫映射

Aider 使用稱為「倉庫地圖」的功能來理解您的專案結構：

- **分析檔案關係**：了解哪些檔案之間如何相依
- **識別關鍵定義**：映射類別、函式和變數定義
- **建立上下文地圖**：為 LLM 提供相關上下文
- **智慧上下文管理**：只發送必要的資訊以保持在 token 限制內

### 工作流程

1. **啟動 Aider**：在儲存庫目錄中執行 `aider`
2. **新增檔案**：告訴 Aider 哪些檔案與您的任務相關
3. **描述任務**：使用自然語言解釋您想完成什麼
4. **檢閱變更**：Aider 顯示建議的程式碼變更
5. **接受/拒絕**：決定是否接受每項變更
6. **自動提交**：接受的變更會自動提交

## 支援的 LLM

Aider 支援多種語言模型：

| 提供者 | 模型 | 備註 |
|----------|-------|-------|
| Anthropic | Claude 3.5 Sonnet | 整體表現最佳 |
| OpenAI | GPT-4o | 表現優異 |
| OpenAI | GPT-4 Turbo | 可靠 |
| Google | Gemini Pro | 良好 |
| OpenRouter | 各種模型 | 靈活選擇 |
| 本地模型 | Ollama 等 | 隱私選項 |

## 命令列選項

Aider 提供了許多命令列選項：

```bash
# 基本使用
aider                          # 啟動互動式工作階段

# 指定檔案
aider file1.py file2.py        # 從特定檔案開始

# 指定模型
aider --model claude-3-5-sonnet-20241022

# 非互動式模式
aider --message "Add error handling to all functions" file.py

# 其他有用的選項
aider --dark-mode              # 使用深色模式
aider --verbose                # 啟用詳細輸出
aider --no-auto-commits        # 停用自動提交
aider --dry-run                # 顯示變更但不套用
```

## 互動式命令

在 Aider 工作階段中，您可以使用以下命令：

| 命令 | 描述 |
|---------|-------------|
| `/add <file>` | 新增檔案到聊天 |
| `/drop <file>` | 從聊天中移除檔案 |
| `/clear` | 清除聊天歷史 |
| `/undo` | 復原最後一次變更 |
| `/diff` | 顯示當前變更的差異 |
| `/help` | 顯示幫助資訊 |
| `/map` | 顯示儲存庫地圖 |
| `/model` | 切換 LLM 模型 |
| `/quit` | 結束 Aider |

## 優點與限制

### 優點

- **無縫的 Git 整合**：每次變更都會自動提交
- **上下文感知**：理解程式碼庫結構和關係
- **互動式且直覺**：自然的對話式介面
- **多功能**：適用於幾乎任何程式設計語言
- **透明化**：提交前會顯示所有建議的變更

### 限制

- **命令列介面**：與 GUI 工具相比，可能無法吸引所有開發者
- **Token 使用量**：大型程式碼庫可能會消耗大量 token
- **複雜重構**：對於大規模、複雜的重構可能需要指導
- **錯誤恢復**：雖然提供 `/undo`，但複雜的錯誤狀態可能難以恢復

## 總結

Aider 是一個強大的命令列工具，為開發者提供了一個與 AI 進行結對程式設計的有效方式。其深度的 Git 整合、程式碼庫映射能力和互動式介面，使其成為希望將 AI 輔助融入其現有工作流程的開發者的絕佳選擇。下一節將提供如何有效使用 Aider 的實際範例。
