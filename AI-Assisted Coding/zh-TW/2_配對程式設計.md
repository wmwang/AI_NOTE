# 2 配對程式設計

配對程式設計意味著您與人工智慧（AI）工具一起建立程式碼。用類似「我的任務是這樣那樣，給我解決方案！」這樣的語句提示 AI 工具很少是個好主意。這只適用於簡單的問題或相對較短的獨立函式。

本書的大部分章節都是為已經相當精通其技藝的程式設計師或開發者而寫的。因此，我們假設您至少熟悉一種程式語言，並且知道如何使用函式、方法和類別。理想情況下，您還擁有一些 IT 背景知識，例如，您可以使用資料庫或 Git。

我們在本章中對這個規則做了例外，因為我們專門針對那些正在學習程式語言的人。在範例中，我們展示了 AI 工具如何幫助您。

但儘管我們在本章中專注於學習過程，配對程式設計當然也適合專業人士！您的前置知識越豐富，專案越複雜，逐步進行就越重要；也就是說，指定程式的結構，並在 AI 的幫助下一次解決或改進一個細節。因此，本章的目的不是向您展示國際銀行帳號（IBAN）驗證函式或數獨求解器如何運作。相反，這些範例應用程式旨在為您提供合理使用 AI 工具的範例。我們想向您展示日常工作中的提示工程是什麼樣的。

我們的範例使用了廣泛使用的 Python 和 Java 語言。但如果您正在學習另一種語言，這不重要！這裡提出的提示在其他語言中同樣有效。正如本書的其他章節一樣，主要不是關於 AI 工具提供的答案（我們通常根本沒有提出或只以縮寫形式提出），而是關於您如何以最有效的方式向 AI 工具提問。

> **理想的 AI 系統**
> 我們在本章的範例中全程使用了帶有 GPT-4o 的 ChatGPT。然而，您同樣可以使用競爭的聊天系統之一。在撰寫本書時，我們使用 Anthropic 的 Claude 有特別好的體驗（https://claude.ai/chat）。在我們 2025 年初的測試中，Claude 不僅提供了優質的答案，而且比較不冗長。較短的答案通常更容易消化，特別是如果您對程式設計的了解有限。儘管 ChatGPT 目前是最知名的，但您可以看到有令人興奮的替代方案——如果不大量使用的話——也是免費的。

如果您正在學習程式設計並且不想盡快完成任務，那麼我們強烈建議不要使用像 GitHub Copilot 這樣的程式碼精靈。典型的練習如「撰寫一個測試參數是否為質數的函式」或「開發一個映射銀行帳戶的類別」對程式碼精靈來說是微不足道的。您只需輸入一個合理的函式名稱如 `testIfPrime`，AI 助理就會回傳整個程式碼。這意味著您很快就完成了，但學到的很少。

基於聊天的系統更適合學習。理想情況下，您應該在聊天歷史的第一個提示中明確說明您是一個需要一些幫助和思考方向但不需要完成程式碼的初學者。

## 2.1 將程式碼結構化為函式

即使我們在本章中不假設有多年的程式設計經驗，我們也不是從零開始的。這有其他書籍可以參考。我們假設您已經知道什麼是變數、分支、迴圈等等，並且能夠處理這些語言建構。

經驗表明，有了這個程度的知識，下一個障礙是如何以有意義的方式將較長的程式碼結構化為函式。這就是我們在這裡開始的地方。

一個起點是簡單地將您撰寫的（義大利麵）程式碼交給您的 AI 聊天夥伴，並詢問它是否看到了將程式碼分割成多個函式的方法。

**提示：** I wrote this Python code to estimate the quality of a password. Can you help me break the code down into functions? Please explain why and how.
[50 行程式碼]

我們可以告訴您：程式碼之後幾乎總是看起來更整潔。ChatGPT 還建議了有意義的函式和參數名稱，並負責程式碼的文件（如果您忽略了它的話）。用您自己的程式碼試試前面的提示或類似的表述！（順便說一下，我們在第 4 章中更詳細地討論了在 AI 幫助下改進現有程式碼。）

### 2.1.2 定義函式和參數

第二個範例的起點是一個練習。您應該程式設計一個從列表中移除重複項的函式。您的第一個問題不太是程式碼本身，而是函式的定義和由此產生的資料流。顯然，您會將要處理的列表作為參數傳遞給函式。但函式應該如何回傳結果？

在提示中，您再次總結任務並指定您正在使用哪種程式語言。同樣，這裡重要的是您對程式碼不感興趣，而是對函式宣告和程式碼結構的協助感興趣。

**提示：** I need to write a Python function to remove duplicates from a list. How should I define the function? Don't provide actual code for the function.

> - **移除重複項的方法：** 您可以使用不同的方法來移除重複項，例如使用集合或遍歷列表並檢查現有元素。考慮您想要如何處理元素的順序（保留或不保留）。
> - **邊緣情況：** 考慮邊緣情況，如空列表或所有元素都相同的列表。

### 2.1.3 關於函式和參數的詳細問題

在處理函式和參數時，有許多特殊情況，這些情況因程式語言而異。當然，您可以閱讀線上文件或 Python 書籍，但您也可以簡單地詢問 AI 工具。如果您不想閱讀冗長的離題內容，您應該要求簡潔的答案表述。特別是 ChatGPT 有在其回應中冗長的傾向，並提供比您需要的更多細節。

不要忘記在提示中包含您正在使用的程式語言名稱。（對於後續問題您不必這樣做，在這種情況下，語言模型已經知道正確的上下文。）

第一個問題涉及經常出現的問題，即函式應該回傳多部分結果。再次注意提示：為了讓 AI 工具明白我們在說什麼，我們還提供了一個簡單的範例。

**提示：** Can my Python functions return more than one data item (for example, a string and a number)?

> 是的，您的 Python 函式可以回傳多個資料項，例如字串和數字，使用多種方法。以下是最常見的方式：
> ```python
> # 使用逗號分隔的值列表來建立 tuple
> def get_info():
>     return "Hello", 42
>
> # 回傳包含值的列表
> def get_info():
>     return ["Hello", 42]
> ```
> [另外三個變體：字典、namedtuple、自訂物件]

對於程式設計初學者來說，關於宣告函式參數列表的問題同樣多。您可以像對同學或導師一樣向您選擇的 AI 工具詢問這些問題。

**提示：** Using Python, how can I define a function to pass a variable number of parameters? And how can I process those parameters inside the function?

**提示：** I want to use optional parameters with default values. What's the Python way to do this?

**提示：** How should I name functions and parameters?

**提示：** Within a Python function, can I access/change variables defined outside?

**提示：** What is the recommended order for a function with several parameters?

ChatGPT 在這類問題上表現最佳，會一頁接一頁地描述每個可以想到的變體和特殊情況。如果您沒有時間或興趣閱讀這些內容，您需要讓 AI 工具明白您只對基礎知識感興趣，而且您還在程式設計的起步階段。

**提示：** Once again, please. Be concise, drop special cases. I am a Python beginner.

### 2.1.4 程式碼最佳化

您可能已經有可以運作的程式碼，只是想要最佳化。ChatGPT 也可以在這方面幫助您。但是，請確保在每個步驟之後檢查您的程式是否仍按預期運作！

前兩個提示是關於將現有程式碼適應通常的 Python 規則。ChatGPT 已經知道正確縮排以及變數和函式命名等約定。只有在您有特殊要求時才需要深入這些細節。理想情況下，您應該提供範本和具體的說明。

**提示：** Please adapt this code to the Python naming conventions.
[100 行程式碼]

**提示：** Could you find better names for the variables and functions used in this code?

> 好的範例：`total_sales`、`current_temperature`、`user_input_calculated`。
> 不要使用：`TotalSales`、`CurrentTemperature`、`UserInputCalculated`。

許多程式設計師在註解程式碼方面相當粗心。AI 工具也可以接管這項不受歡迎的工作（另見第 6 章）。在最簡單的情況下，如第一個範例中的微型提示就足夠了。

**提示：** Please add comments to this code.

根據誰要閱讀您的程式碼和註解，您應該指定這個目標群體。但請注意，雖然 AI 工具通常可以理解您的程式碼，但它無法猜測您為什麼決定採用特定的方法。因此，AI 註解只能部分取代您自己的註解。

**提示：** Add comments to this file to make the code understandable to a novice programmer.

> **提示**
> 事後註解程式碼也可以在您從同事、網際網路或其他來源收到程式碼且不太理解時提供幫助。

也許您開發的函式可以運作但非常慢或佔用大量記憶體。如果您要求 AI 工具最佳化您的程式碼，您應該盡可能具體地說明需要最佳化的內容（例如速度）。

**提示：** This function is rather slow. Do you see any way to make the code more efficient?

**提示：** When processing large files, this function uses far too much memory. Is there a way to reduce the memory consumption of the code?

### 2.1.5 程式碼重複使用

在學習程式語言時，AI 工具不應該是您唯一的輔助工具。如果您在網際網路上搜尋程式碼並遇到您不理解的建構，您可以讓它們為您解釋。（正如您永遠不應該盲目接受 AI 產生的程式碼，除非您真正理解它，這個規則也適用於來自 Stack Overflow 或其他知名網站的程式碼。程式碼可能實際上是正確的，但適用於與您所想的完全不同的應用。）

以下提示展示了如何將網際網路作為通用顧問的優勢與 AI 的優勢結合起來。在這裡，AI 工具應該解釋之前您不知道的 `linalg.norm` 函式。

**提示：** This function apparently calculates the distance between some points. Can you explain what `np.linalg.norm()` does?

```python
def get_distances(points_xy):
    l1 = np.linalg.norm(
      points_xy[0, :] - points_xy[1, :])
    l2 = np.linalg.norm(
      points_xy[1, :] - points_xy[2, :])
    return l1, l2
```

您可能正在從一種語言切換到另一種語言，或者您可能在網際網路上找到了另一種語言的解決方案。大多數 AI 工具都能將您的程式碼從一種語言翻譯成另一種語言。

**提示：** Last year I wrote a PowerShell script to download the image of a web page. How can I rewrite this code using Python?
[20 行 PowerShell 程式碼]

根據我們的經驗，這種程式碼轉換並不總是完美運作。每種程式語言、每個函式庫都有自己的特性。花時間徹底測試程式碼並仔細檢查其功能。如果您想學習程式設計且重點不在於快速完成任務，更好的做法是將 AI 機器人建議的模組或函式庫作為自己研究的起點，然後自己開發程式碼。
