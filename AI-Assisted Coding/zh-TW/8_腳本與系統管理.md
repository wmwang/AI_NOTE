# 8 腳本與系統管理

在程式設計領域中，很少有像使用 Python 或 Bash 撰寫腳本時那樣，AI 輔助（如果正確使用！）能發揮如此好的效果。在這種情境下，腳本的特性扮演了關鍵角色。通常，相對較少的程式碼行數就足以完成一個明確定義的任務。雖然大型程式設計專案的程式碼分散在數十個、甚至數百個檔案中，但腳本通常可以在單一檔案中表達。因此，幾乎沒有什麼依賴關係或上下文問題會讓 AI 語言模型難以應付。

我們原本預期在 Windows 上使用 PowerShell 撰寫腳本也會有類似好的體驗。但結果卻相當令人失望。我們測試中使用的 GPT-4o 模型（截至 2024 年中）產出的程式碼品質差了很多，且有明顯更多的錯誤。當然，網路上可供 PowerShell 訓練的資料比 Python 少，但差距如此明顯令人驚訝。

儘管如此，在本章中，我們展示了各種 ChatGPT、GitHub Copilot 和其他 AI 工具幫助我們的範例。同樣地，我們的目標是向您展示廣泛的可能應用。也許有一兩個您還沒想到的想法？

在本章中，我們也稍微超越腳本的範疇，探討正規表達式和系統管理等次要主題。

系統管理並不真的是傳統的程式設計主題，因此不在本書的重點範圍內。但在現實生活中，許多開發人員和程式設計師也負責設定 Docker 容器、Git 儲存庫、資料庫系統和伺服器。ChatGPT 和其他 AI 工具完全能夠協助處理這些工作。

## 8.1 腳本

從我們的角度來看，撰寫新腳本的理想工作環境是具有 AI 輔助的編輯器（例如 Visual Studio Code [VS Code] 加上 GitHub Copilot）和一個專門的聊天視窗——無論是作為編輯器的一部分，還是在網頁瀏覽器中獨立開啟。AI 聊天提供了腳本各部分基本設計和研究所用命令、cmdlet 或模組的協助。編輯器則支援程式碼輸入、文件撰寫和除錯。以下幾頁展示了豐富多樣的範例。

### 8.1.1 Bash：遍歷文字檔

您想在 Bash 腳本中處理一個文字檔，並跳過第一行的欄位標籤。最簡單的方法是什麼？問 AI 工具！

**提示：** Bash：我想要遍歷檔案的所有行，但跳過第一行。

在 Bash 中，如果您想要遍歷檔案的所有行但跳過第一行，可以使用以下腳本：

```bash
# 假設 'file.txt' 是您的檔案
tail -n +2 file.txt | while IFS= read -r line
do
  echo "$line"
done
```

### 8.1.2 PowerShell：照片排序

您想使用 PowerShell 腳本分析整批照片的可交換圖檔格式 (EXIF) 資料。圖片檔案應該被移動到以 yyyy-mm 命名的目錄中，例如 2024-06 表示 2024 年 6 月拍攝的照片。

**提示：** PowerShell 指令碼：我有大量 JPEG 檔案需要整理。我想讀取 EXIF 日期，然後將檔案移動到 yyyy-mm（年/月）格式的資料夾中。沒有 EXIF 資料的檔案應移動到特殊資料夾。你能幫我嗎？

當然！以下是一個 PowerShell 腳本，它會讀取 JPEG 檔案的 EXIF 日期，並根據年份和月份將它們整理到資料夾中。沒有 EXIF 資料的檔案將被移動到單獨的資料夾。

指令碼和說明...

ChatGPT 提供了一個大約 60 行的腳本，使用 System.Drawing 系統函式庫來分析 EXIF 資料。雖然這個解決方案相對繁瑣，但有一個很大的優點：不需要安裝額外的工具或函式庫。

然而，初次測試執行失敗了：腳本會開啟每個圖片檔案進行 EXIF 分析，但不會再次關閉檔案。這就是為什麼 Move-Item 對每個檔案都會出現錯誤訊息。

原始腳本還有一個缺陷：它只考慮了 `*.jpg` 檔案，而沒有考慮 `*.jpeg` 檔案。因此，您需要請 ChatGPT 進行修正。

**提示：** Move-Item 導致錯誤：「The process cannot access the file because it is being used by another process.」我猜腳本需要在移動檔案之前先關閉它。也請處理 `*.jpeg` 檔案。

您說得對。圖片檔案應該在移動之前關閉，以避免「檔案正在使用中」的錯誤。此外，我們也會加入對 `*.jpeg` 檔案的處理。以下是更新後的腳本：

```powershell
...
$jpegFiles = Get-ChildItem -Path $sourceDir `
                           -Filter *.jpg, *.jpeg
...
```

ChatGPT 認出了它的錯誤並修正了腳本，但立刻插入了新的錯誤：`-Filter` 選項期望單一字元字串作為引數，而不是列舉。

**提示：** `$jpegFiles = Get-ChildItem -Path $sourceDir -Filter *.jpg, *.jpeg` 導致錯誤：Cannot convert System.Object[] to the type System.String required by parameter Filter.

我為這個錯誤道歉。`-Filter` 參數只接受單一字串。我們可以透過使用兩次 `Get-ChildItem` 或使用帶有 `-Recurse` 的 `-Include` 來處理 `.jpg` 和 `.jpeg` 檔案。以下是修正後的腳本：

```powershell
...
$jpegFiles = Get-ChildItem -Path $sourceDir `
                           -Include *.jpg, *.jpeg -Recurse
...
```

只有在第三次嘗試時提供的腳本才能令人滿意地運作。現在使用的 `-Recurse` 選項有一個額外的好處，即 PowerShell 不僅處理圖片目錄，還會處理所有子目錄。這種行為實際上應該是不言自明的，但誠然在原始提示中並沒有被要求。

這個範例令人困惑的是錯誤的瑣碎性。雖然腳本的基本概念在第一次嘗試時就沒問題，但其中的錯誤在某種程度上是初學者的錯誤。這削弱了對 AI 工具的信任。

### 8.1.3 Python：SQLite 應用

您想在 Python 腳本中使用 SQLite 管理內部資料。您知道 SQLite 是一個用於小型資料庫應用的極簡函式庫，但您從未將 SQLite 與 Python 一起使用過。

**提示：** 我正在開發一個 Python 腳本，想使用 SQLite 作為本機資料庫。我應該使用哪個模組？需要安裝額外的函式庫嗎？可以提供一個簡短的 Hello World 範例嗎？

要在 Python 中使用 SQLite，您可以使用內建的 sqlite3 模組。不需要安裝額外的函式庫，因為 sqlite3 已包含在 Python 的標準函式庫中。此模組提供了一個輕量的磁碟型資料庫，不需要單獨的伺服器處理程序，並允許使用非標準的 SQL 變體進行存取。

以下是一個使用 Python 操作 SQLite 的簡短「Hello World」範例：

```python
import sqlite3

conn = sqlite3.connect(':memory:')
cursor = conn.cursor()

cursor.execute("""
    CREATE TABLE hello_world 
                  (id INTEGER PRIMARY KEY, message TEXT)
""")
cursor.execute("""
    INSERT INTO hello_world (message) 
                  VALUES ('Hello, World!')
""")
conn.commit()

cursor.execute('SELECT * FROM hello_world')
print(cursor.fetchall())  # 輸出: [(1, 'Hello, World!')]

conn.close()
```

### 8.1.4 Bash 疑難排解

ChatGPT 也能協助疑難排解。前陣子，我們想要處理一個逗號分隔值 (CSV) 檔案。URL 和資料庫的兩個欄位以分號分隔。我們嘗試使用 `cut -d;` 來分割欄位，但這會失敗：

**提示：** 以下 Bash 腳本無法運作。為什麼？

```bash
tail -n +2 unused.csv | while IFS= read -r line; do
    url=$(echo $line | cut -d; -f1)
    db=$(echo $line | cut -d; -f2)
    echo "url=$url db=$db"
done
```

ChatGPT 版本 4 及更高版本能立即識別錯誤。（您呢？）

您的腳本問題可能是因為 `cut` 命令中的分隔符號處理不正確。分隔符號需要正確指定，特殊字元（如您的情況中的分號 `;`）應該加上引號，以確保 shell 能正確解釋它們。

以下是修正程式碼的方法：

```bash
tail -n +2 unused.csv | while IFS= read -r line; do
    url=$(echo "$line" | cut -d';' -f1)
    db=$(echo "$line" | cut -d';' -f2)
    echo "url=$url db=$db"
done
```

較舊的 GPT 版本在這個範例中失敗了，本機模型 llama3:8b 和 gemma2:9b 也是如此。

### 8.1.5 Python：遞迴搜尋檔案

這個範例的起點是開發一個腳本，用來遞迴處理所有具有預定義副檔名的檔案，從一個起始目錄開始（另見第 10 章，第 10.4 節）。

**提示：** Python：我需要一個函式，能遞迴地回傳目錄中所有具有給定副檔名的檔案。我會傳遞兩個參數：目錄路徑和副檔名列表（例如 `[".js", ".php"]`）。程式碼應該能在 Linux、macOS 和 Linux 上執行。能幫我嗎？

當然！以下是一個 Python 函式，它會遞迴搜尋目錄並回傳所有匹配給定副檔名的檔案列表。此函式使用 `os` 和 `os.path` 模組進行目錄遍歷。

```python
def get_files_with_extensions(directory, extensions):
    matching_files = []
    for root, _, files in os.walk(directory):
        for file in files:
            if any(file.endswith(ext) for ext in extensions):
                matching_files.append(os.path.join(root, file))
    return matching_files

# 範例用法
print(get_files_with_extensions("/home/kofler",
                                [".js", ".php"]))
```
