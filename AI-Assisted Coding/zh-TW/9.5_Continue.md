# Continue

Continue 是一個領先的開源 AI 程式碼助手，可連接到任何您選擇的 LLM。它作為 VS Code 和 JetBrains IDE 的擴充功能運作，提供直接的上下文感知程式碼輔助。

## 為什麼使用 Continue？

- **開源**：完全開源，擁有活躍的社群
- **模型靈活性**：連接到 Ollama、OpenAI、Anthropic、Google 等
- **IDE 整合**：適用於 VS Code 和 JetBrains IDE
- **上下文感知**：使用您的程式碼庫作為程式碼輔助的上下文
- **可自訂**：配置提示、行為和整合
- **Tab 自動補全**：在您輸入時提供智慧程式碼建議

## 安裝

### VS Code

1. 開啟 VS Code
2. 前往擴充功能面板（`Ctrl+Shift+X` 或 `Cmd+Shift+X`）
3. 搜尋 "Continue"
4. 點擊安裝

或者從命令列安裝：

```bash
code --install-extension continue.continue
```

### JetBrains IDE

1. 開啟您的 JetBrains IDE（IntelliJ、PyCharm、WebStorm 等）
2. 前往 **設定 → 外掛**
3. 搜尋 "Continue"
4. 點擊安裝

## 初始設定

當您第一次安裝 Continue 時，設定嚮導將會幫助您：

1. **選擇您的模型提供者**：選擇 Ollama、OpenAI 等
2. **配置連接**：提供必要的 API 端點和金鑰
3. **選擇自動補全模型**：選擇一個模型用於行內建議
4. **選擇嵌入模型**：用於程式碼庫索引

### config.json 設定

Continue 使用一個配置檔案（`~/.continue/config.json`），您可以在其中定義所有設定：

```json
{
  "models": [
    {
      "title": "Ollama CodeLlama",
      "provider": "ollama",
      "model": "codellama:13b"
    },
    {
      "title": "Ollama DeepSeek Coder",
      "provider": "ollama",
      "model": "deepseek-coder:6.7b"
    },
    {
      "title": "GPT-4",
      "provider": "openai",
      "model": "gpt-4"
    }
  ],
  "tabAutocompleteModel": {
    "title": "Tab Autocomplete",
    "provider": "ollama",
    "model": "deepseek-coder:6.7b"
  },
  "embeddingsProvider": {
    "provider": "ollama",
    "model": "nomic-embed-text"
  },
  "allowAnonymousTelemetry": false
}
```

## 核心功能

### 側邊欄聊天

主要的聊天介面作為 IDE 側邊欄面板運作：

- **上下文注入**：使用 `@` 提及文件、符號或終端機內容
- **程式碼高亮**：回應使用適當的語法高亮
- **應用程式碼**：點擊即可將建議的程式碼變更應用到您的編輯器
- **對話歷史**：保留所有對話的記錄

### Tab 自動補全

Continue 在您輸入時提供智慧行內補全：

- **行補全**：完成單行或多行程式碼
- **函數補全**：生成整個函數實作
- **上下文感知**：使用周圍的程式碼作為上下文
- **多語言**：支援數十種程式語言

### 內嵌聊天

高亮程式碼並直接在編輯器中開啟聊天：

- **解釋**：高亮程式碼並要求解釋
- **重構**：選擇程式碼並要求改進
- **除錯**：選擇有問題的程式碼並要求修復
- **文件**：為選定的程式碼生成文件

### 命令

使用斜線命令執行常見操作：

- `/edit`：內嵌編輯選定的程式碼
- `/comment`：為程式碼添加文件註解
- `/debug`：除錯選定的程式碼
- `/share`：匯出對話以供分享

## 使用 Ollama 配置 Continue

### 基本配置

在您的 `~/.continue/config.json` 中：

```json
{
  "models": [
    {
      "title": "Ollama DeepSeek Coder",
      "provider": "ollama",
      "model": "deepseek-coder:6.7b"
    },
    {
      "title": "Ollama CodeLlama",
      "provider": "ollama",
      "model": "codellama:13b"
    },
    {
      "title": "Ollama Llama 3",
      "provider": "ollama",
      "model": "llama3"
    }
  ],
  "tabAutocompleteModel": {
    "title": "Autocomplete",
    "provider": "ollama",
    "model": "deepseek-coder:6.7b"
  },
  "embeddingsProvider": {
    "provider": "ollama",
    "model": "nomic-embed-text"
  }
}
```

### 推薦用於不同用途的模型

| 用途 | 推薦模型 | 大小 | 原因 |
|-----------|------------------|----------|-----------|
| 聊天/助手 | DeepSeek Coder 6.7B | 6.7B | 最佳品質/速度比 |
| 自動補全 | DeepSeek Coder 1.3B | 1.3B | 補全速度快 |
| 程式碼審查 | CodeLlama 13B | 13B | 良好的推理能力 |
| 通用 | Llama 3 8B | 8B | 通用能力強 |
| 嵌入 | Nomic Embed Text | 274M | 高品質嵌入 |

### 多模型設定

您可以同時配置多個模型並在它們之間切換：

```json
{
  "models": [
    {
      "title": "Fast (DeepSeek 1.3B)",
      "provider": "ollama",
      "model": "deepseek-coder:1.3b",
      "contextLength": 4096
    },
    {
      "title": "Balanced (DeepSeek 6.7B)",
      "provider": "ollama",
      "model": "deepseek-coder:6.7b",
      "contextLength": 16384
    },
    {
      "title": "Powerful (CodeLlama 34B)",
      "provider": "ollama",
      "model": "codellama:34b",
      "contextLength": 16384
    }
  ]
}
```

## 進階功能

### 自訂提示

在您的配置中建立自訂提示：

```json
{
  "customCommands": [
    {
      "name": "review",
      "description": "Review code for issues",
      "prompt": "Please review the following code for bugs, performance issues, and best practices:\n\n{{{ input }}}"
    },
    {
      "name": "document",
      "description": "Add documentation",
      "prompt": "Add comprehensive documentation to this code including docstrings and type hints:\n\n{{{ input }}}"
    }
  ]

```

### 程式碼庫上下文

Continue 可以索引您的整個程式碼庫：

- **語意搜尋**：使用嵌入來尋找相關程式碼
- **文件提及**：使用 `@filename` 來包含特定文件
- **程式碼定義**：引用函數和類別定義
- **終端機上下文**：包含終端機輸出

### 上下文提供者

使用 `@` 提及新增上下文：

- `@file`：引用特定文件
- `@folder`：包含資料夾內容
- `@codebase`：搜尋相關程式碼
- `@terminal`：使用終端機輸出
- `@url`：從 URL 獲取內容
- `@docs`：查詢文件網站

### 實驗性功能

- **程式碼路徑索引**：索引和查詢程式碼依賴關係
- **多檔案編輯**：跨多個檔案建議變更
- **測試執行**：從聊天執行測試
- **文件生成**：自動化文件生成

## 故障排除

### 自動補全不運作

1. 確保 `tabAutocompleteModel` 已配置
2. 確保補全模型正在運行（`ollama list`）
3. 檢查模型是否足夠小以實現快速補全（建議 ≤ 7B）
4. 重啟 IDE

### 模型連接問題

1. 驗證 Ollama 是否正在運行：`ollama list`
2. 確保模型已拉取：`ollama pull <model-name>`
3. 檢查 config.json 語法
4. 查看 Continue 輸出面板中的錯誤

### 效能緩慢

1. 使用較小的模型進行自動補全（1.3B - 3B）
2. 使用較大的模型進行聊天（7B - 13B）
3. 確保 GPU 加速已啟用
4. 調整 `contextLength` 以平衡效能與品質
