# Ollama

Ollama 是一個輕量級、易於安裝的工具，用於在本地運行大型語言模型（LLM）。它簡化了在您自己的機器上下載和運行模型的過程，對於希望在不依賴外部 API 的情況下進行實驗的開發者來說非常理想。

## 為什麼使用 Ollama？

- **隱私與安全**：您的數據保留在本地機器上，永遠不會發送到外部伺服器。
- **離線能力**：下載模型後，不需要網路連接。
- **成本效益**：沒有 API 費用——只需下載並運行。
- **效能**：直接在您的硬體上運行，充分利用 GPU 和 CPU 的能力。
- **多樣的模型選擇**：開源模型的大型庫，包括程式碼專用模型。

## 安裝

### Linux 和 macOS（WSL2）

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

### macOS

從 [ollama.com](https://ollama.com) 下載應用程式。

### Linux

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

### Docker

```bash
docker run -d -v ollama:/root/.ollama -p 11434:11434 --name ollama ollama/ollama
```

### Windows

從 [ollama.com](https://ollama.com) 下載 Windows 安裝程式，或透過 WSL2 使用 Linux 安裝方法。

## 基本用法

### 下載模型

要下載模型，請使用 `pull` 命令：

```bash
# 下載程式碼專用模型
ollama pull codellama

# 下載通用聊天模型
ollama pull llama3

# 下載不同的模型大小
ollama pull codellama:7b
ollama pull codellama:13b
ollama pull codellama:34b
```

### 互動式聊天

直接與模型開始聊天會話：

```bash
ollama run codellama
```

這將開啟一個互動式提示，您可以輸入提示並即時獲得回應。

### 單次提示

對於單一提示，使用 `run` 命令並帶有提示：

```bash
ollama run codellama "Write a Python function to sort a list"
```

### 列出可用模型

要查看您已下載的模型：

```bash
ollama list
```

### 刪除模型

要移除不再需要的模型：

```bash
ollama rm codellama:7b
```

## 程式碼專用模型

Ollama 支援多種為程式碼生成和理解設計的模型：

| 模型 | 參數 | 最佳用途 |
|-------|------------|-----------|
| CodeLlama | 7B, 13B, 34B | 通用程式碼生成 |
| DeepSeek Coder | 1.3B, 6.7B, 33B | 程式碼補全與聊天 |
| CodeQwen | 1.5B, 7B | 程式碼理解和生成 |
| StarCoder2 | 3B, 7B, 15B | 程式碼補全 |
| Phind-CodeLlama | 34B | 程式碼推理 |
| WizardCoder | 7B, 13B, 34B | 指令遵循的程式碼生成 |

### 推薦用於程式碼助手

對於程式碼補全和助手使用，以下模型最有效：

- **DeepSeek Coder**：整體最佳的程式碼生成品質
- **CodeQwen**：良好的效能/大小比
- **CodeLlama**：穩定可靠的通用選擇
- **StarCoder2**：適合快速補全

## 進階配置

### Modelfile（自訂模型）

創建一個 `Modelfile` 以自訂模型行為：

```bash
# 創建 Modelfile
cat > Modelfile <<EOF
FROM codellama:13b

# 設定溫度（創造力）
PARAMETER temperature 0.7

# 設定系統提示
SYSTEM You are an expert Python developer. Always provide well-documented code with type hints.
EOF

# 從 Modelfile 創建自訂模型
ollama create my-python-assistant -f Modelfile

# 運行自訂模型
ollama run my-python-assistant
```

### 多模態模型

Ollama 支援多模態模型，可以處理圖像輸入：

```bash
# 下載多模態模型
ollama pull llava

# 運行圖像分析
ollama run llava "What's in this image? /path/to/image.jpg"
```

### REST API

Ollama 自動提供 REST API 服務：

```bash
# 生成回應
curl http://localhost:11434/api/generate -d '{
  "model": "codellama",
  "prompt": "Write a Python function to calculate fibonacci numbers",
  "stream": false
}'

# 聊天 API
curl http://localhost:11434/api/chat -d '{
  "model": "codellama",
  "messages": [
    {"role": "user", "content": "Explain decorators in Python"}
  ],
  "stream": false
}'
```

## 效能調整

### GPU 加速

Ollama 自動偵測並使用可用的 GPU：

- **NVIDIA**：需要 CUDA 支援
- **AMD**：需要 ROCm 支援
- **Apple Silicon**：透過 Metal 框架最佳化

### 記憶體管理

要調整分配給模型的記憶體：

```bash
# 檢查模型資訊
ollama show codellama:13b

# 保持模型在記憶體中載入
ollama keep codellama:13b
```

### 量化

Ollama 使用 4-bit 量化來減少記憶體使用，同時保持效能。不同的量化級別可用：

- **q4_0**：最快，最低品質
- **q4_1**：品質略有提升
- **q5_0**：品質與速度的良好平衡
- **q8_0**：最高品質，更多記憶體使用

## 搭配 AI 輔助程式設計工具

Ollama 作為多種程式碼助手和 IDE 擴充功能的後端：

- **Continue**：用於 VS Code 和 JetBrains 的開源 IDE 擴充功能
- **Open WebUI**：網頁式聊天介面
- **Tabby**：自託管的 AI 程式碼補全工具
- **Cursor**：AI 驅動的程式碼編輯器
- **aider**：命令列 AI 程式設計助手

### 典型設定流程

1. 安裝並啟動 Ollama
2. 下載適當的程式碼模型
3. 配置您選擇的工具以使用 Ollama 作為後端
4. 開始在您的 IDE 中使用 AI 輔助

## 故障排除

### 常見問題

**模型下載緩慢**
- 嘗試較小的模型變體
- 檢查網路連接
- 使用鏡像站點（如果有）

**記憶體不足錯誤**
- 嘗試量化較多的模型
- 關閉其他應用程式以釋放 RAM
- 嘗試較小的模型（例如 7B 而不是 13B）

**生成速度慢**
- 確保啟用 GPU 加速
- 嘗試較小的模型
- 檢查系統資源使用情況

### 有用的命令

```bash
# 檢查 Ollama 狀態
ollama --version

# 查看正在運行的模型
ollama ps

# 更新模型到最新版本
ollama pull codellama
```

## 總結

Ollama 讓本地 LLM 的強大功能在程式碼輔助方面變得易於使用。只需最少的設定，您就可以在自己的機器上運行最先進的程式碼模型，確保隱私，同時提供與雲端服務相當的效能。結合程式碼助手工具，它為 AI 輔助程式設計提供了一個完整的、自託管的解決方案。
