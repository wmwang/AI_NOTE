# 3.2 應用程式開發

以下除錯範例涉及 Flutter 函式庫，它允許您為 iOS 和 Android 以及 Web 應用程式建立應用程式。

## 3.2.1 Flutter 與 Null Safety 值

我們使用 Flutter 函式庫建立的行動應用程式需要支援多語言。作為一個現代化的工具組，Flutter 自然提供了相應的工具，但我們確實遇到了一個小問題：應用程式無法再編譯，因為翻譯條目不是 null safe 的。簡單來說，編譯器無法在編譯時確保翻譯物件在執行時是可用的。

dart 編譯器的詳細錯誤訊息已經清楚地指出了該怎麼做，但我們也想讓 AI 工具解決這個問題：

**提示：** Flutter: My app isn't compiling after I inserted translations for labels. ERROR: lib/widgets/team.dart:43:62: Error: Property 'warning' cannot be accessed on 'AppLocalizations?' because it is potentially null. Try accessing using ?. instead. title: Text(AppLocalizations.of(context).warning),

> 這個錯誤是因為 AppLocalizations.of(context) 呼叫可能回傳 null，而 Dart 的 null safety 功能阻止您在可能為 null 的物件上存取 warning 屬性。要修復此問題，您可以使用 null-aware 運算子 ?。以下是修改程式碼的方式：
> ```dart
> title: Text(AppLocalizations.of(context)?.warning ?? 'Warning')
> ```

從錯誤訊息可以看出，應該對 AppLocalizations 使用 null-aware 運算子（即 `?.`）。然而，AI 工具更進一步，附加了替換文字 'Warning'（雙問號在 Flutter/Dart 中為 null 值提供後備值）。如果沒有這個更改，應用程式仍然無法編譯，因為周圍的 `Text()` 函式期望一個字串，而不是 null。

AI 工具也簡要解釋了更改的背景，我們覺得非常有幫助：

> 此更改做了兩件事：
> 1. 使用 `?.` 運算子安全地存取 warning 屬性——僅當 AppLocalizations.of(context) 不為 null 時。
> 2. 使用 `??` 運算子提供後備字串（在本例中為 'Warning'），以防 AppLocalizations.of(context) 為 null 或 warning 不可用。

我們再次對 AI 工具的除錯能力感到驚喜。

## 3.2.2 Flutter 中的佈局

使用 Flutter 函式庫進行應用程式開發需要對 Web 開發進行一些重新思考。如果區域沒有被 widget 完全填滿，或者如果它們超出了可用空間，Flutter 會用黑黃條紋背景填充該區域以指示問題。這幫助我們識別問題，但遺憾的是不能幫助我們解決問題。讓我們看看 AI 是否也能在這裡幫助我們：

> **圖 3.4** Flutter 在底邊緣顯示的錯誤佈局

**提示：** Flutter: I get a yellow-black striped area at the bottom of my app. The widget I am using looks like this:
```dart
return Scaffold(
  drawer: const MyDrawer(),
  body: Column(
    children: <Widget>[
```

> 應用程式底部的黑黃條紋區域通常表示 Flutter 中的溢出錯誤。當 widget 嘗試渲染不適合其分配空間的內容時就會發生這種情況。在您的案例中，問題可能是由 Scaffold 主體中的 Column widget 引起的。[更多細節]

Claude 3.5 Sonnet 接下來解釋了兩個選項，兩者都能達到期望的結果。生成的程式碼可以直接替換且正常運作。

關於兩種不同解決方案（一個使用 ListView，一個使用 SingleChildScrollView）的簡短解釋讓我們不清楚哪個更適合我們的應用程式。因此，我們詢問：

**提示：** What is the difference between a ListView and a SingleChildScrollView?

> ListView 和 SingleChildScrollView 都是 Flutter 中可滾動的 widget，但它們有一些關鍵區別：
>
> 1. 子 widget：
>    - ListView 設計用於高效處理可能大量數量的子項。它可以有多個子 widget。
>    - SingleChildScrollView 只能有一個子 widget，通常是 Column 或 Row。
> 2. 延遲載入：
>    - ListView 支援延遲載入。它只建構目前在螢幕上可見的子 widget，對於長列表更節省記憶體。
>    - SingleChildScrollView 一次性建構所有內容，對於大量內容可能效率較低。
> [更多細節]

接著是簡潔描述差異的額外要點。因此，AI 在這裡也沒有讓我們失望：僅在閱讀第一個要點後，我們就意識到使用 SingleChildScrollView 的第二種變體更適合我們過大的元素。
