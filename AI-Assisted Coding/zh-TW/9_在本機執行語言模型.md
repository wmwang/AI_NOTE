# 9 在本機執行語言模型

自從 ChatGPT 取得突破性成功以來，大型語言模型（LLM）一直是最熱門的話題。無論是 Gemini、Claude 還是 ChatGPT，它們都有一個共同點：它們由公司提供，而這些公司甚至免費提供其模型的介面。作為 Pro 用戶，您只需支付相對適中的費用，即可獲得某些功能，例如優先處理查詢或使用最新、最好的模型變體。

無論您選擇哪種方案，您都在使用託管在雲端中的服務，因此您無法確定透過提示傳送給這些服務的資料會發生什麼事。對這些線上服務的另一個批評是，您依賴於各自的供應商。目前有足夠多的可供選擇，因此如果某個供應商撤回其服務，您不會有問題，但依賴性仍然存在。

特別是對隱私和對單一供應商依賴的擔憂，促使一些人和公司尋找使用語言模型的替代方案。這就是為什麼對在本機執行 LLM 的興趣日益增長。本章揭示這種方式的運作效果：

- 首先，我們將提供免費可用 LLM 的概述。我們也會探討您的硬體需要多強大才能以合理的速度在本機執行語言模型。
- 然後，我們描述免費的 GPT4All 和 Ollama 程式的安裝和使用。這兩個程式相容於多種 LLM，甚至在產生回應時可以考慮本機文件。GPT4All 面向初學者，而 Ollama 為許多專業應用提供了良好的基礎，從程式開發到應用程式介面（API）的使用。
- Ollama 實際上是一個純文字導向的工具。然而，您可以將該程式與 Open WebUI 介面結合使用，然後以類似網頁版 ChatGPT 等的方式執行提示。
- Continue 外掛實現了一個程式碼精靈，您可以用它來代替 GitHub Copilot 或其他商業服務。Continue 能夠與本機 Ollama 安裝或外部 LLM API 通訊。
- Ollama 也可以透過 API 使用。這讓您可以編寫自己的程式，這些程式不依賴商業 LLM，而是使用在本機執行的語言模型。
- Tabby 專案代表了 Ollama 和 Continue 組合的替代方案：它由一個 LLM 伺服器（可安裝在自己的電腦或公司強大的工作站上）和一個具有程式碼精靈功能的編輯器外掛組成。

## 9.1 大型語言模型的選擇困難

如果您涉足生成式 AI，不可避免地會接觸到各種 LLM。本節旨在為您提供不同模型系列、其預期用途以及本機模型所需資源的粗略概述。

### 9.1.1 哪些模型解決哪些問題？

一般來說，您需要區分 LLM 的商業模型和免費可用模型。商業模型由供應商作為服務提供。ChatGPT 只是這種服務的一個例子。在基本版本中，您可以免費在瀏覽器中使用 ChatGPT 進行查詢。OpenAI 是 ChatGPT 背後的供應商，提供月費訂閱方案，具有多項好處，例如存取更好的模型、更高的訊息量或更早存取新功能。Anthropic 以 Claude 追求類似的策略，Google 以 Gemini 也是如此。

除了熟悉的瀏覽器介面或應用程式外，您也可以透過 API 使用這些模型。這裡的計費模式通常不同，您需要按 token 付費，即輸入和輸出 token 都要付費。每個 token 的費用通常非常低。例如，您可以從 OpenAI 以 $2.50 購買一百萬個輸入 token，以 $10 購買一百萬個輸出 token。較小的 GPT-4o Mini 模型一百萬個輸入 token 為 $0.15，一百萬個輸出 token 為 $0.60（截至 2025 年 1 月）。

免費可用模型（如 Llama、Gemma 或 Mistral）的本機版本沒有成本。但是，您現在必須提供執行模型的基礎設施。因此，您應該仔細權衡選擇哪種方案。您通常可以透過模型的品質和回應時間來控制成本。

目前市場上有許多不同的 LLM。有些模型共用相同的架構，可以歸為同一系列。透過 GPT、Llama、Mistral 和 Claude，您將在以下列表中了解一些範例及其各自的特點：

**GPT：OpenAI 的生成式預訓練 Transformer**

最著名的模型可能屬於 OpenAI 的 GPT 系列。它們構成了 ChatGPT 的基礎。GPT 基於 Transformer 模型，並使用針對文字生成最佳化的特殊變體。這些模型成功的關鍵在於使用大量文字進行全面訓練和後續的微調。

OpenAI 透過自己的服務提供 GPT 模型。其使用部分免費、部分收費。GPT 系列的模型可以是多模態的；也就是說，它們不僅能處理文字，還能處理圖片或音訊。遺憾的是，OpenAI 的 LLM 檔案無法免費下載。

典型任務包括聊天機器人、內容生成和程式碼生成。

**Llama：Meta AI 的大型語言模型**

來自 Meta 的 Llama 語言模型使用與 GPT 類似的架構，基於 Transformer 架構。由於各種最佳化和設計決策，這些模型更注重在較小模型大小下的效率和性能。

Meta 的 Llama 模型受 Llama 社群授權協議約束。此授權適用於私人使用和研究，但限制了模型的商業使用。

典型任務包括自然語言處理（NLP）任務，如文字生成、翻譯和研究任務。Code Llama 是針對程式開發任務最佳化的 Llama 變體。

**Mistral/Mixtral**

Mistral 和 Mixtral 模型由 Mistral 製作，這是一家專注於 AI 的法國公司。該公司以其開源模型聞名，這些模型受 Apache 2 授權約束。除此之外，Mistral 還透過自己的 API 提供其他付費模型。

Mixtral 是 Mistral 最新的模型，是混合專家語言模型（MoE-LLM），它結合了多個專家模型，從而實現更好的性能和高度準確性。與 GPT 和 Llama 不同，Mixtral 不是完整的 Transformer 模型，而是純解碼器模型。

典型任務包括摘要文字、結構化、回答問題和程式碼補全。

**Claude**

Claude 模型由 Anthropic 開發。開發的重點不僅在於模型的性能，還在於安全性和倫理。

Claude 模型本質上也基於 Transformer 架構。區別在於憲法 AI（Constitutional AI），這是一種基於監督學習和強化學習的特殊訓練方法。

現代 Claude 模型的另一個特點是其 200,000 token 的上下文大小，與競爭對手相比非常大。遺憾的是，與 OpenAI 一樣，Anthropic 不提供其模型的下載選項。

典型任務包括商業應用、複雜分析、大量資料摘要和程式開發。

對於哪種語言模型為哪種應用提供理想結果的客觀比較是極其困難的。目前還沒有公認的標準，但有許多專案和科學研究。閱讀以下針對程式開發應用精選的資源，您可以獲得初步印象：

- https://lmarena.ai/?leaderboard
- https://artificialanalysis.ai/models/gpt-4o/providers
- https://github.com/continuedev/what-llm-to-use
- https://evalplus.github.io/leaderboard.html
- https://symflower.com/en/company/blog/2024/comparing-llm-benchmarks

最終，您必須自己實驗，找出哪個 LLM 在您的硬體上最適合您的需求。

> **免費還是真正的開源？**
> 「免費」的 LLM（如 Llama、Gemma、StarCoder 或 Mistral）受單獨的授權約束，這些授權可能限制其使用。大多數 LLM 比您在 Linux 世界中習慣的有更多限制。
>
> 特別是在您開始商業使用免費 LLM 之前，您必須熟悉使用條款。這些條款可以在各自的專案頁面和 https://huggingface.co 上各個語言模型的描述中找到。

### 9.1.2 硬體需求

能夠免費下載許多語言模型是很好的，但使用 GPT4All 或 Ollama 等程式在本機執行這些模型往往因為高硬體需求而失敗。例如，如果您嘗試在標準電腦上以最佳品質的 405B 版本執行 Llama 3.1 模型，那根本不可能。因此，我們現在來看看執行 LLM 的一般硬體需求：

**CPU**
您應該至少擁有第 11 代 Intel 晶片或基於 Zen4 的 AMD CPU。Apple 的 M 系列 CPU 也非常適合。比核心數量更重要的是處理器的功能集，特別是 AI 模型所需的加速矩陣乘法。

**RAM**
對於 7B 模型，您的系統應該至少有 16 GB 的 RAM。（7B 表示 70 億個參數。）

**硬碟空間**
您應該預留大約 50 GB 的可用硬碟空間，以便儲存幾個模型來試用。

**圖形處理單元（GPU）**
雖然顯示卡不是絕對必要的，但它可以大幅加速模型的執行。但是，整個模型必須儲存在顯示卡的視訊記憶體中。對於 4 位元量化的模型，您應該預期以下 VRAM 大小：

- 7B：約 4 GB VRAM
- 13B：約 8 GB VRAM
- 30B：約 16 GB VRAM
- 65B：約 32 GB VRAM

Apple ARM CPU（Apple Silicon）的電腦是特殊情況。它們的 GPU 可以存取整個工作記憶體，因此不需要區分 RAM 和 VRAM。

這些建議來自 Justin Hayes，他是 Open WebUI 和 Ollama 的貢獻者之一，可在 Open WebUI 的 GitHub issue 中找到：

https://github.com/open-webui/open-webui/discussions/736#discussioncomment-8474297

只要您的平台提供足夠的記憶體來載入相應的模型，您就可以執行它。但是，如果您使用的硬體太弱，必須預期模型的回饋會非常慢。最好使用一個能在您的系統上順暢運作的稍小模型，而不是一個勉強能在系統上運行的模型。

> **速度基準測試**
> 評估本機語言模型執行效果如何的決定性參數是每秒可以處理（輸入）或生成（輸出）的 token 數量。對於程式開發中的互動使用，您的本機語言模型應該至少達到 30 tokens/s（輸出）。
>
> Token 速率主要取決於兩件事：模型大小和可用硬體。在以下網頁上，您可以找到 Windows、Linux 和 macOS 上各種 LLM 和硬體配置的結果：
>
> - https://llm.aidatatools.com/results-windows.php
> - https://llm.aidatatools.com/results-linux.php
> - https://llm.aidatatools.com/results-macos.php

### 9.1.3 本機執行模型的替代方案

除了本機執行 AI 模型和使用商業模型外，還有另一個中間步驟。您也可以在雲端租用帶有 GPU 的虛擬機器。所有主要供應商都提供這種服務。

但請注意，使用此類虛擬機器有時可能會產生相當可觀的費用，因此您應該仔細考慮是否只是偶爾需要它們。以下概述展示了主要雲端供應商此類平台的選項：

- **Amazon Web Services (AWS)：EC2 GPU Instance**
  https://aws.amazon.com/ec2/instance-types/#Accelerated_Computing
- **Google Cloud Platform (GCP)：Cloud GPU**
  https://cloud.google.com/gpu
- **Microsoft Azure：NVIDIA Deep Learning (ND) 或 NVIDIA Compute (NC) 系列計算實例**
  https://learn.microsoft.com/en-us/azure/virtual-machines/sizes/overview
- **NVIDIA：GPU 雲端運算**
  www.nvidia.com/en-us/data-center/gpu-cloud-computing/

各供應商提供不同大小的雲端實例。價格很大程度上取決於各自的配置和期限。例如，如果您租用 Microsoft Azure 的 NCasT4_v3 實例，每月大約需要 $500。作為回報，您可獲得 28 GB RAM、176 GB 硬碟空間和一個具有 16 GB VRAM 的 NVIDIA Tesla T4 GPU。

如果您選擇例如 NC48ads_A100_v4 實例，可以進一步提升此配置。此實例提供 880 GB RAM、96 核心、3,832 臨時儲存空間和 4 個 NVIDIA A100 GPU（每個具有 80 GB VRAM）。但此配置的費用上升到每月大約 $14,000！

如需精確的價格計算，您應該查閱各供應商的價格計算器。無論您選擇哪個商業供應商，都會產生費用。另一個選擇是本機執行，無論是在您自己的電腦上還是在本地網路中。為此，您可以使用 GPT4All 或 Ollama 等平台，它們讓執行模型變得更容易。

> **Groq**
> Groq 公司（不要與 xAI 的 AI 聊天機器人 Grok 混淆！）以其 GroqCloud 在 OpenAI 或 Anthropic 等商業 AI API 供應商和 AWS 等純雲端/硬體供應商之間走了一條中間路線。Groq 讓您透過 API 金鑰執行 LLM 提示。可以選擇各種不同大小的開源 LLM。計費基於使用量（具有不同價格的 token 數量，取決於模型）。Groq 使用內部開發的 AI 加速器（Language Processing Unit），可以特別快速地執行 LLM。您可以在 https://groq.com/groqcloud 找到關於此主題的資訊。
