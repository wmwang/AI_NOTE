# 8.4 系統管理

身為開發者，您的主要職責當然是程式開發。然而在現實中，系統管理與軟體開發之間的界線有時會變得模糊。您需要設定 Docker 容器、網路配置、虛擬機器或資料庫伺服器（即使只是作為測試環境）。如下面的範例所示，AI 工具可以提供清晰的指示來協助您。

我們不想將本節命名為「Linux 系統管理」，但必須承認接下來的內容主要與 Linux 相關。這不（僅僅）是出於個人偏好，主要是因為 Linux 系統管理是在文字模式下進行的。您在終端機中執行命令、修改設定檔等等。語言模型在這方面可以提供很好的幫助。

相比之下，Windows 的系統管理通常是在圖形使用者介面中進行的。這不符合語言模型的特性。此外，截至 2025 年初，ChatGPT 等工具尚無法透過對話框或設定精靈的截圖來輔助說明。

## 8.4.1 SSH 金鑰認證

第一個範例是關於透過 scp 自動將檔案複製到 Linux 伺服器。必須使用 Secure Shell (SSH) 金鑰，以避免每次執行指令碼時都需要輸入密碼。

**提示：** 我想使用 scp 將檔案備份到 Linux 伺服器。指令碼應該能在無需互動的情況下執行。有什麼安全的方式可以達成這個目標？

若要使用 scp 將檔案備份到 Linux 伺服器且無需互動，您可以使用基於金鑰的認證。以下是設定的逐步指南。

首先，在本機上產生一組 SSH 金鑰對：

```
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
```

按照提示操作。當被詢問密碼短語時，您可以選擇留空以實現自動化流程，但建議使用密碼短語以增加安全性。金鑰對通常會儲存在 `~/.ssh/id_rsa` 和 `~/.ssh/id_rsa.pub` 中。

使用 `ssh-copy-id` 將您的公鑰複製到遠端伺服器：

```
ssh-copy-id username@remote_server_ip
```

此命令會提示您輸入遠端伺服器上的密碼。成功執行後，您的公鑰將被加入到遠端伺服器上的 `~/.ssh/authorized_keys` 檔案中。

驗證您可以在不提示輸入密碼的情況下連線到遠端伺服器：

```
ssh username@remote_server_ip
```

ChatGPT 接著提供了一個簡單備份指令碼的範例及其透過 Cron 自動化的方式。我們在這裡刪除了輸出的這部分內容。

基本上，如果指令碼要在 Windows 上執行，SSH 金鑰認證也可以運作。但是設定會稍微複雜一些。ChatGPT 說明了如何手動在伺服器上設定公鑰。

**提示：** 我的備份指令碼應該在 Windows 上執行。但是 ssh-copy-id 無法使用。我可以繞過這個問題嗎？

是的，您可以在 Windows 上透過手動將 SSH 公鑰複製到遠端伺服器來繞過這個問題。首先，使用 Windows 10/11 內建的 OpenSSH 用戶端產生一組 SSH 金鑰對。

```
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
```

按照提示儲存金鑰（預設會儲存在 `C:\Users\your_username\.ssh\id_rsa` 和 `C:\Users\your_username\.ssh\id_rsa.pub`）。

由於 Windows 上沒有 `ssh-copy-id`，您需要手動將公鑰檔案（`id_rsa.pub`）的內容複製到遠端伺服器上的 `~/.ssh/authorized_keys` 檔案中。用文字編輯器開啟 `id_rsa.pub` 檔案並複製其內容。使用 SSH 用戶端連線到遠端伺服器。登入後，開啟或建立遠端伺服器上的 `~/.ssh/authorized_keys` 檔案：

```
nano ~/.ssh/authorized_keys
```

將複製的公鑰貼入 `authorized_keys` 檔案並儲存。

## 8.4.2 Docker 設定

為了測試資料庫用戶端，您需要一個本機 MySQL 伺服器。您不想進行「真正」的安裝，而是想使用 Docker。這該怎麼做？

**提示：** 我有一個本機程式需要存取 MySQL 資料庫。我該如何設定對應的 Docker 容器？

ChatGPT 提供了詳細的七點說明，從 Docker 安裝開始，到 Python 應用程式範例結束。我們在這裡只摘錄了三個核心的 Docker 命令：

```
docker network create my_network
docker pull mysql:latest
docker run --name my_mysql_container --network my_network \
  -e MYSQL_ROOT_PASSWORD=rootpassword \
  -e MYSQL_DATABASE=mydatabase \
  -e MYSQL_USER=myuser \
  -e MYSQL_PASSWORD=mypassword \
  -p 3306:3306 -d mysql:latest
```

基本上，這是可行的。為容器建立獨立的網路在這裡完全是多餘的，但不會造成干擾。在最後一點中，ChatGPT 指出一旦容器終止，儲存在資料庫中的所有資料都會遺失。為了避免這種情況，您必須在建立容器時建立一個磁碟區。之後您可以重複使用這個磁碟區，將資料庫從一個容器轉移到下一個。

如果您想在容器停止後仍然保存資料，可以掛載一個磁碟區。修改 Docker run 命令以包含磁碟區：


## 8.4.4 修復虛擬機器中網頁伺服器的網路問題

您已經在虛擬機器中安裝了 Linux、網頁伺服器和各種其他工具。但是您無法從主機存取網頁伺服器。ChatGPT 能幫忙嗎？

記住要盡可能精確。哪個虛擬化程式？哪個 Linux 發行版？哪個網頁伺服器？哪個主機作業系統？什麼錯誤訊息？您已經嘗試了什麼來縮小錯誤範圍？

**提示：** 我在 Windows 筆記型電腦上使用 VirtualBox。我設定了一台安裝 Fedora Linux 的虛擬機器。在 Fedora 中，我安裝了 Apache。`systemctl status httpd` 顯示它運行正常。`hostname -I` 顯示我的 IP 位址是 10.0.2.15。然而，在我的主機系統上，我無法存取網站。每當我嘗試開啟 http://10.0.2.15 時，網頁瀏覽器顯示 ERR_CONNECTION_TIMED_OUT。這個錯誤可能是什麼原因造成的？

您遇到的問題可能是由 VirtualBox 設定的網路配置所導致。預設情況下，VirtualBox 通常使用 NAT（網路位址轉換）進行網路連線，這可能會導致從主機存取客體機器的問題。以下是解決此問題的幾個步驟：

- 將網路介面卡變更為橋接介面卡 [更多細節]
- 確定新的 IP 位址 [更多細節]
- 更新防火牆規則。確保 Fedora 上的防火牆允許 HTTP 流量。首先，檢查防火牆狀態：

```
sudo firewall-cmd --state
```

如果防火牆正在運行，允許 HTTP 流量：

```
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --reload
```

- 存取網頁伺服器 [更多細節]

## 8.4.5 Linux 伺服器設定

您想設定一台 Linux 伺服器。您使用 RAID-1（獨立磁碟冗餘陣列 1），也就是兩個內容相互鏡像的固態硬碟（SSD）。如果兩個磁碟機中的一個故障，您不用擔心資料遺失。

現在開機檔案通常只在第一個磁碟上。如果該磁碟故障，您不會遺失任何資料，但將無法再啟動電腦。這可以透過在兩個磁碟上正確設定 GRUB 開機載入器來解決。但這該怎麼做？

**提示：** Linux：我想使用 RAID-1 設定一台有兩個磁碟的伺服器。無論是第一個還是第二個磁碟故障，伺服器都應該能夠開機。你能幫忙嗎？

ChatGPT 提供了詳細的指示，從磁碟分割開始，然後解釋 RAID 設定，最後涵蓋 GRUB 的安裝。我們在這裡只重現了對問題至關重要的部分：

確保 GRUB 已安裝在兩個磁碟上。您可以在安裝後手動執行：

```
sudo grub-install /dev/sda
sudo grub-install /dev/sdb
sudo update-grub
```

因此，GRUB 開機載入器必須安裝在兩個磁碟上，這通常不是預設情況。請注意，對於使用 NVMe（非揮發性記憶體快遞）SSD 的現代伺服器，您必須使用裝置名稱 `dev/nvme0n1` 和 `/dev/nvme1n1`，而不是 `/dev/sda` 和 `/dev/sdb`。
