## 6.5 過時的文件

很少有什麼事比過時的文件更糟糕。如果您曾經在應用程式中遇到無法信任文件（例如函式的註解區塊）的問題，您在未來會花更多時間進行調查，並且在必要時，還會額外查看程式碼。因此，擴充功能的實作將需要更多時間。

過時的文件是如何產生的？通常，這並不是因為有人惡意想誤導他人，而是多半因為時間壓力，或者只是忘記更新文件。然而，留下的是信任的喪失，因此文件的價值也大幅降低。

如果您自己負責更新某個函式，可以借助 AI 工具自動更新文件。`fetchUserData` 函式就是這類工作流程的一個範例。它最初的實作方式是傳入使用者 ID 和回呼函式。該函式本身從網頁伺服器載入資料，然後呼叫回呼函式。經過修改後，該函式現在實作為非同步函式。它只接受使用者的 ID，並回傳一個包含已載入使用者資料的 Promise 物件。然而，函式的文件仍然參照舊的實作。在程式碼中如下所示：

```javascript
/**
 * Fetches user data from the server using the Fetch 
 API.
 *
 * This function sends a GET request to the server to 
 retrieve 
 * user data based on the provided user ID. The 
 response is 
 * expected to be in JSON format.
 *

 * @param {number} userId The ID of the user whose 
 data 
 *                        is to be fetched.
 * @param {function} callback A callback function that 
 will
 *                            be called with the user 
 data.
 *                            The callback should 
 accept two
 *                            parameters: an error (if 
 any) or
 *                            null if the request was 
 successful
 *                            and the user data.
 *
 * @return {void}
 */
async function fetchUserData(userId: number) {
  const response = await 
fetch(`/api/users/${userId}`);
  if (!response.ok) {
    throw new Error('Failed to fetch user data');
  }
  return response.json();
}  
```

函式的描述仍然是正確的，但參數和回傳值的說明是不正確的。要解決這個問題，例如，您可以再次使用 VS Code 的 GitHub Copilot 整合，並提交以下提示：

**提示：** Please update the documentation for the fetchUserData function.

```javascript
/**
 * Fetches user data from the server.
 *
 * @param userId - The ID of the user to fetch data 
 for.
 * @returns A promise that resolves to the user data.
 * @throws An error if the request fails.
 */

async function fetchUserData(userId: number) {
  const response = await 
fetch(`/api/users/${userId}`);
  if (!response.ok) {
    throw new Error("Failed to fetch user data");
  }
  return response.json();
} 
```

這裡立即值得注意的是，Copilot 介入了註解的結構。原始的詳細函式描述已被移除，參數類型也被省略了。總體而言，結果是正確的，但有些資訊遺失了。這是一個很好的範例，說明當提示過於簡單時，AI 工具不一定能改善情況。您可以修改提示，讓結果更接近原始版本。然而，這只有在更廣泛的原始碼中才值得，因為在這種情況下，您在提示上花費的時間可能足以讓您自己撰寫文件。

另一種選擇是使用其他工具，例如 ChatGPT。使用相同的提示，結果會有所不同：

**提示：** Please update the documentation for the following function:
[code of the function]

 returned as

ChatGPT 沒有刪除任何資訊，而是調整了參數的文件，並新增了一個如何使用該函式的範例。您還會收到一份所做變更的清單，包括相關的解釋。

使用 AI 工具時，您還必須權衡工具所提供的便利性與其他工具的品質。例如，GitHub Copilot 允許您直接在開發環境中修改程式碼，而其他工具可能需要您將原始碼貼到另一個命令提示中。
