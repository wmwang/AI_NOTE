# 5.6 端對端測試

理論上，像 Vitest 這樣的測試框架可以讓你覆蓋測試金字塔的所有層級。這類框架最適合用於撰寫單元測試和整合測試。對於端對端測試，則有專門的框架，如 Selenium、Cypress 或 Playwright，它們專注於讓你盡可能方便地實作端對端測試。

在接下來的章節中，你將學習如何使用 Playwright 為一個簡單的登入表單生成端對端測試。該表單有一個使用者名稱輸入欄位和一個密碼輸入欄位，還有一個用於提交表單的按鈕。表單透過 JavaScript 函式傳送。成功登入後，使用者會被重新導向到 `/list` 路徑。表單的 HTML 程式碼如下：

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta
      name="viewport"
      content="width=device-width, initial-scale=1.0"
    />
    <title>Login Form</title>
  </head>
  <body>
    <div class="login-container">
      <h2>Login</h2>
      <form id="loginForm">
        <input
          type="text"
          id="username"
          name="username"
          placeholder="Username"
          required
        />
        <input
          type="password"
          id="password"
          name="password"
          placeholder="Password"
          required
        />
        <button type="submit">Login</button>
      </form>
    </div>
  </body>
</html>
```

我們在這裡刻意省略了樣式和 JavaScript，因為端對端測試是一種經典的黑盒測試，從使用者的角度檢查功能。

## 5.6.1 生成端對端測試

讓端對端測試正常運作的前提是安裝 Playwright，你可以透過 `npx init playwright@latest` 命令來完成安裝。你可以使用 `npx playwright test` 命令來執行 Playwright 測試。然而，安裝後立即只有範例測試可用。但你可以透過讓 ChatGPT 為登入表單生成測試來改變這一點。

為了讓測試正常運作，AI 工具需要你提供一些資訊。你必須提供 HTML 結構，並指定表單可以透過哪個位址存取。你還必須指定應使用哪個使用者名稱和密碼，以及成功登入後會被重新導向到哪裡。對應的提示可能如下：

**提示：** 為以下 HTML 結構建立一個 Playwright 測試。假設有 JavaScript 處理表單提交。表單位於 http://localhost:3000/login。使用「admin」作為使用者名稱，「test」作為密碼。成功登入後，使用者將被重新導向到 http://localhost:3000/list。

[HTML 表單程式碼]

以下是為提供的 HTML 結構撰寫的 Playwright 測試。這個測試腳本將分別在使用者名稱和密碼欄位中填入「admin」和「test」，提交表單，並檢查使用者是否被重新導向到 http://localhost:3000/list。

首先，確保你已安裝 Playwright。如果尚未安裝，你可以使用以下命令安裝：

```
npm install @playwright/test
```

接下來，建立一個測試檔案（例如 `login.test.js`）並加入以下程式碼：

```javascript
// @playwright/test module
import { test, expect } from "@playwright/test";
test("login test", async ({ page }) => {
  // 前往登入頁面
  await page.goto("http://localhost:3000/login");
  // 填入使用者名稱
  await page.fill("#username", "admin");
  // 填入密碼
  await page.fill("#password", "test");
  // 點擊登入按鈕
  await page.click('button[type="submit"]');
  // 檢查重新導向的 URL 是否為列表頁面
  await page.waitForURL("http://localhost:3000/list");
  // 檢查新的 URL 是否為列表頁面
  expect(page.url()).toBe("http://localhost:3000/list");
});
```

[程式碼說明]

在你的應用程式的測試目錄中建立一個名為 `login.test.js` 的新檔案，然後按照提示回應中的建議執行測試命令。如果測試成功執行，它會檢查你的登入流程。ChatGPT 還為每一行提供了註解，讓你可以輕鬆追蹤步驟。你可以在以下的說明中進一步了解各個命令的含義。

你必須接受的一個限制——至少對於像 ChatGPT 這樣的 LLM 來說——是你無法在一個步驟中用一個端對端測試覆蓋整個工作流程。問題在於 AI 工具無法動態運行由多個視圖組成的工作流程。因此，你需要逐步進行，然後將各個部分組合成一個完整的工作流程。

## 5.6.2 建立頁面物件

如果你使用頁面物件（page object），你的測試會更加清晰。頁面物件為你提供了語意化的方法，讓你能夠與頁面互動。這些方法可以對頁面進行讀取和寫入存取，例如，回傳一個特定的值用於檢查，或將值寫入表單欄位。這使得測試程式碼更容易閱讀。

要在端對端測試中使用頁面物件，請使用之前的 ChatGPT 對話，並將以下內容作為下一個提示：

**提示：** 你能在測試中加入頁面物件嗎？

當然可以！使用頁面物件模型（Page Object Model, POM）有助於更有效率地組織和維護你的測試程式碼。以下是一個包含登入頁面頁面物件並將其整合到測試中的範例。

首先，為頁面物件建立一個名為 `LoginPage.js` 的新檔案。

```javascript
// LoginPage.js
export class LoginPage {
  /**
   * @param {import('@playwright/test').Page} page
   */
  constructor(page) {
    this.page = page;
    this.usernameInput = '#username';
    this.passwordInput = '#password';
    this.submitButton = 'button[type="submit"]';
  }
  async goto() {
    await this.page.goto('http://localhost:3000/login');
  }
  async login(username, password) {
    await this.page.fill(this.usernameInput, username);
    await this.page.fill(this.passwordInput, password);
    await this.page.click(this.submitButton);
  }
}
```

接下來，更新測試檔案以使用此頁面物件。

```javascript
// login.test.js
import { test, expect } from '@playwright/test';
import { LoginPage } from './LoginPage';
test('login test', async ({ page }) => {
  const loginPage = new LoginPage(page);
  // 導航到登入頁面
  await loginPage.goto();
  // 執行登入操作
  await loginPage.login('admin', 'test');
  // 檢查重新導向的 URL 是否為列表頁面
  await page.waitForURL("http://localhost:3000/list");
  // 檢查新的 URL 是否為列表頁面
  expect(page.url()).toBe('http://localhost:3000/list');
});
```

[程式碼說明]

如果純粹從行數來看，使用頁面物件並沒有給你帶來什麼好處。填寫表單時省下的三行程式碼，又被匯入、實例化頁面物件和呼叫方法所消耗掉了。然而，在更大的測試中，程式碼通常會更加緊湊和可讀。你還可以在不同的端對端測試之間重複使用頁面物件，從而獲益。例如，如果你將登入流程提取到這樣一個頁面物件中，你就可以在每個需要登入的端對端測試中使用它。
