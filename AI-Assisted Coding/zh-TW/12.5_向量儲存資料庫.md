# 12.5 向量儲存資料庫

為了避免每次啟動應用程式時都要重新計算向量嵌入，我們會將索引儲存在資料庫中。向量資料庫的任務是儲存非結構化資料的數值表示（具有各個維度的向量），並使用各種演算法進行搜尋。這些實作分為：由外部函式庫在現有（關聯式或 NoSQL）資料庫中執行搜尋的類型、在資料庫中使用外掛程式的類型，以及真正的向量資料庫。

真正的向量資料庫與 SQL 和 NoSQL 資料庫處於同等地位。SQL 資料庫將來源資料強制轉換為嚴格的結構（透過具有許多資料類型的固定表格），NoSQL 僅透過鍵值對（資料類型如數字、字串或日期）對資料進行粗略結構化。對於向量資料庫，來源資料以非結構化方式儲存，搜尋時僅使用計算出的向量。

真正的向量資料庫的一個傑出代表是 Milvus。該資料庫在免費的 Apache 2 授權條款下提供，可以在不同場景中非常輕鬆地使用：Lite 版本非常適合原型設計或硬體規格非常低的裝置。對於生產部署，Milvus Standalone 可以在 Docker 容器中非常輕鬆地啟動。如果你想大規模部署，也可以在 Kubernetes 叢集中以雲原生架構啟動 Milvus。此外，你還可以從開發公司 Zilliz 租用 Milvus 作為現成的雲端解決方案。

我們嘗試了 Lite 版本和獨立版本。兩種變體在我們非常小的資料集中都表現令人信服。使用 compose.yaml 檔案的 Docker 設定幾乎可以一鍵啟動。無論你使用 Lite/獨立版本還是 Milvus 的雲端版本，API 呼叫都沒有區別。這讓你能在幾秒鐘內將小型測試專案轉變為可擴展的解決方案。

我們在 Docker 設定中使用 compose 來運行 Milvus 的獨立版本作為我們的 RAG 專案。如果你已經安裝了最新版本的 Docker 和 compose 外掛，你需要做的就是從 GitHub 下載提供的設定檔並透過 docker compose up -d 啟動設定：

https://github.com/milvus-io/milvus/releases/download/v2.4.6/milvus-standalone-docker-compose.yml

除了資料庫之外，還會啟動兩個其他服務：etcd 設定服務和 MinIO 物件儲存。資料庫稍後會在連接埠 19530 上可用，無需認證。

Zilliz 還開發了一個圖形使用者介面（GUI）作為開源軟體來管理資料庫，顯示已索引的 PDF 文件和網頁。螢幕左側是三個集合，每個都是用不同的嵌入模型建立的（見圖 12.3）。

> 圖 12.3 Attu 顯示 Milvus 向量資料庫

Attu 可以作為額外的容器輕鬆添加到 Docker Compose 設定中。只需將以下幾行添加到下載的 Compose 檔案的 services 部分：

```yaml
attu:
  image: zilliz/attu:v2.4
  environment:
    MILVUS_URL: milvus-standalone:19530
  ports:
    - "8000:3000"  
```

使用 docker compose up -d 啟動容器後，你可以在 http://localhost:8000 或啟動 docker compose 設定的主機名稱上存取網頁介面。我們發現深入了解向量資料庫的內部資料結構非常有趣。

要為我們的範例專案使用 Milvus 資料庫，我們在建立 VectorStoreIndex 時包含了一個 StorageContext LlamaIndex：

```python
from llama_index.vector_stores.milvus import MilvusVectorStore
from llama_index.embeddings.ollama import OllamaEmbedding
from llama_index.core import StorageContext
vector_store = MilvusVectorStore(uri="http://localhost:19530",
                                 dim=1024, 
                                 overwrite=True)
storage_context = StorageContext.from_defaults(
  vector_store=vector_store
)
index = VectorStoreIndex.from_documents(
    documents=documents,
    transformations=[
        SentenceSplitter(),
        TitleExtractor(),
        OllamaEmbedding(model_name="mxbai-embed-large"),
    ],
    storage_context=storage_context,
    show_progress=True,
)  
```

除了 uri（指之前在 Docker 中啟動的資料庫）之外，還必須指定向量的維度，然後必須與使用的嵌入模型相匹配。這裡使用的 mxbai-embed-large 模型使用具有 1,024 個維度的向量。為了避免重複，overwrite=True 參數會在建立索引之前刪除資料庫。應用程式現已準備就緒，我們可以使用索引內容嘗試第一批自然語言查詢。
