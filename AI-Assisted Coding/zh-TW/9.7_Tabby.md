# Tabby

Tabby 是一個開源、可自託管的 AI 程式碼補全工具，專為團隊使用而設計。與依賴雲端服務的程式碼助手不同，Tabby 運行在您自己的基礎設施上，提供隱私和對您的開發工作流程的控制。

## 為什麼使用 Tabby？

- **自託管**：在您自己的伺服器或本地機器上運行
- **隱私**：程式碼永遠不會離開您的基礎設施
- **團隊導向**：為團隊協作和共享模型而設計
- **快速補全**：為低延遲的行內建議而最佳化
- **多 IDE 支援**：適用於 VS Code、JetBrains、Vim 等
- **可擴展**：支援多種後端模型和硬體配置

## 安裝

### Docker 安裝（推薦）

使用 Docker 是運行 Tabby 最簡單的方式：

```bash
docker run -it --gpus all \
  -p 8080:8080 \
  -v $HOME/.tabby:/data \
  tabbyml/tabby
```

### 不使用 GPU

```bash
docker run -it \
  -p 8080:8080 \
  -v $HOME/.tabby:/data \
  tabbyml/tabby
```

### 使用 Docker Compose

創建一個 `docker-compose.yml`：

```yaml
version: '3.8'
services:
  tabby:
    image: tabbyml/tabby
    ports:
      - "8080:8080"
    volumes:
      - tabby-data:/data
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: all
              capabilities: [gpu]

volumes:
  tabby-data:
```

### 從二進位檔案安裝

```bash
# Linux/macOS
curl -sSL https://tabby.tabbyml.com/install.sh | sh

# 或者下載特定版本
# 從 https://github.com/TabbyML/tabby/releases 下載
```

## 初始設定

1. **啟動 Tabby 伺服器**：運行 Docker 容器或二進位檔案
2. **存取管理介面**：導航到 `http://localhost:8080`
3. **配置模型**：選擇並下載補全模型
4. **安裝 IDE 擴充功能**：在您喜歡的編輯器中安裝 Tabby 擴充功能
5. **連接到伺服器**：配置擴充功能以指向您的 Tabby 實例

### 配置

Tabby 使用一個配置檔案進行設定。基本配置：

```json
{
  "server": {
    "address": "0.0.0.0",
    "port": 8080
  },
  "model": {
    "completion": "TabbyML/StarCoder-3B",
    "chat": "TabbyML/CodeLlama-7B"
  },
  "repositories": [],
  "device": "cuda"
}
```

## 核心功能

### 程式碼補全

Tabby 的主要功能是快速的行內程式碼補全：

- **行補全**：完成當前或下一行
- **函數補全**：建議整個函數實作
- **多行補全**：為程式碼區塊生成多行程式碼
- **上下文感知**：使用周圍的程式碼作為上下文
- **語言支援**：適用於許多程式語言

### 聊天介面

Tabby 包含一個用於程式碼相關問題的聊天介面：

- **程式碼問答**：詢問有關您程式碼庫的問題
- **解釋**：獲取程式碼片段的解釋
- **除錯**：獲得修復錯誤的幫助
- **重構建議**：獲取程式碼改進的想法

### 管理儀表板

基於網頁的管理介面提供：

- **模型管理**：下載和配置模型
- **使用者管理**：管理團隊存取
- **使用統計**：監控補全使用情況
- **健康檢查**：監控伺服器健康狀態
- **配置**：調整設定

## IDE 擴充功能

### VS Code

1. 開啟 VS Code
2. 前往擴充功能面板
3. 搜尋 "Tabby"
4. 安裝 "Tabby" 擴充功能
5. 配置伺服器 URL（預設：`http://localhost:8080`）

### JetBrains IDE

1. 開啟您的 JetBrains IDE
2. 前往 **設定 → 外掛**
3. 搜尋 "Tabby"
4. 安裝外掛
5. 在設定中配置伺服器 URL

### Vim/Neovim

對於 Neovim，使用適當的 LSP 客戶端進行配置：

```lua
-- 使用 nvim-lspconfig 的配置範例
require'lspconfig'.tabby.setup{
  cmd = {'tabby', 'lsp'},
  filetypes = {'python', 'javascript', 'typescript', 'rust', 'go', 'java'},
}
```

## 搭配 Ollama 使用

Tabby 可以使用 Ollama 作為其後端模型提供者：

### 配置

更新 Tabby 的配置以使用 Ollama：

```json
{
  "server": {
    "address": "0.0.0.0",
    "port": 8080
  },
  "model": {
    "completion": "tabbyml/StarCoder-3B",
    "chat": "ollama:codellama:13b"
  }
}
```

或者使用 Ollama 作為獨立的補全引擎，並讓 Tabby 管理 IDE 擴充功能。

### 搭配使用的好處

- **Tabby 的 IDE 整合** 與 **Ollama 的模型生態系統**
- Tabby 管理補全基礎設施
- Ollama 提供模型服務
- 無縫的 IDE 體驗

## 團隊功能

團隊可以共享模型部署：

3. **一致性**：程式碼補全建議的一致性

### 存取控制

- **基於角色的存取**：管理和使用者角色
- **Token 認證**：安全的 API 存取
- **團隊管理**：新增/移除團隊成員

### 分析

- **使用指標**：追蹤補全率
- **採用率**：監控團隊採用情況
- **效能**：追蹤回應延遲

## 進階配置

### GPU 支援

Tabby 支援多種 GPU 配置：

```bash
# NVIDIA GPU
docker run -it --gpus all tabbyml/tabby

# 特定 GPU
docker run -it --gpus '"device=0"' tabbyml/tabby

# 多個 GPU
docker run -it --gpus all tabbyml/tabby
```

### 模型選擇

選擇適合您需求的模型：

| 模型 | 大小 | 延遲 | 最佳用途 |
|-------|------|----------|-----------|
| StarCoder-3B | 3B | 低 | 快速補全 |
| CodeLlama-7B | 7B | 中 | 平衡的補全 |
| DeepSeek-Coder-6.7B | 6.7B | 中 | 高品質補全 |
| CodeLlama-13B | 13B | 高 | 複雜的補全 |

### 自訂模型

您可以使用自訂模型：

```bash
# 使用來自 HuggingFace 的自訂模型
docker run -it --gpus all \
  -p 8080:8080 \
  tabbyml/tabby \
  --model TabbyML/StarCoder-3B \
  --device cuda
```

### 效能調整

- **批次大小**：調整以適應 GPU 記憶體
- **最大 token 數**：配置補全長度
- **快取**：啟用模型快取以提升效能
- **並行**：調整並行請求

## API

Tabby 提供了一個用於整合的 HTTP API：

### 補全端點

```bash
curl -X POST http://localhost:8080/v1/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "TabbyML/StarCoder-3B",
    "prompt": "def fibonacci(n):",
    "max_tokens": 100,
    "temperature": 0.5
  }'
```

### 聊天端點

```bash
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "TabbyML/CodeLlama-7B",
    "messages": [
      {"role": "user", "content": "Explain this code: def foo(): pass"}
    ]
  }'
```

### 健康檢查

```bash
curl http://localhost:8080/health
```

## 部署選項

### 本地開發

運行在您的開發機器上以供個人使用：

```bash
docker run -d -p 8080:8080 -v $HOME/.tabby:/data tabbyml/tabby
```

### 團隊伺服器

部署在團隊可存取的共享伺服器上：

```bash
docker run -d \
  -p 8080:8080 \
  -v /data/tabby:/data \
  --gpus all \
  --restart unless-stopped \
  tabbyml/tabby
```

### 生產部署

對於生產環境：

- 使用 Kubernetes 進行編排
- 設置負載平衡
- 配置 SSL/TLS
- 實施身份驗證
- 監控和日誌記錄

## 故障排除

### 常見問題

**補全不運作**
- 確保 Tabby 伺服器正在運行
- 檢查 IDE 擴充功能配置
- 驗證模型是否已載入
- 查看 Tabby 伺服器日誌

**補全緩慢**
- 如果可用，啟用 GPU 加速
- 嘗試較小的模型
- 檢查伺服器資源
- 調整批次大小

**連接問題**
- 驗證伺服器 URL 和埠
- 檢查防火牆設定
- 確保 Docker 網路已正確配置

### 有用的命令

```bash
# 檢查 Tabby 狀態
curl http://localhost:8080/health

# 查看 Docker 日誌
docker logs tabby

# 重啟 Tabby
docker restart tabby
```
