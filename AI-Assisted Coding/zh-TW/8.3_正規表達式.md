# 8.3 正規表達式

正規表達式是用來描述文字搜尋模式的字串。在這種情況下，必須遵循特定的規則，而這些規則不是很直觀。各種 Linux 命令（如 `grep` 和 `sed`）以及幾乎所有程式語言都支援正規表達式。您可以使用它們從文字中篩選出特定資訊、驗證輸入，或將識別出的模式替換為新的文字。

AI 工具現在可以幫助您處理過去需要耗費大量心力才能編寫的正規表達式。但請注意，正規表達式有數種語法變體，包括可移植作業系統介面 (POSIX)、POSIX 延伸正規表達式 (ERE) 和 Perl 相容正規表達式 (PCRE)。因此，始終指定您使用的工具或程式語言非常重要。ChatGPT 和其他工具就能同時提供應用正規表達式的程式碼。

**提示：** 請給我一個電子郵件地址的正規表達式（PCRE 語法）。

當然，以下是一個匹配電子郵件地址的正規表達式：

```
[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}
```

- `[A-Za-z0-9._%+-]+`：匹配方括號內的一個或多個字元，包括大小寫字母、數字，以及電子郵件地址中常見的某些特殊字元，如句點 (`.`)、底線 (`_`)、百分比 (`%`)、加號 (`+`) 和連字號 (`-`)。
- `@`：匹配 `@` 符號。
- `[A-Za-z0-9.-]+`：匹配一個或多個大小寫字母、數字，以及網域名稱中常見的某些特殊字元，如句點 (`.`) 和連字號 (`-`)。
- `\.`：匹配一個句點（點），因為句點在正規表達式中是特殊字元，所以用反斜線跳脫。
- `[A-Za-z]{2,}`：匹配兩個或多個大小寫字母，用於頂級網域 (TLD)。這確保電子郵件地址的 TLD 部分至少由兩個字母組成。

這個正規表達式模式應該能涵蓋大多數有效的電子郵件地址，但值得注意的是，使用正規表達式驗證電子郵件地址可能相當複雜，因為有各種標準和可能的邊界情況。

也可以更簡短。然而，在第二個範例中請求的較簡單表達式精度較低。

**提示：** 可以使用更短的正規表達式嗎？

是的，更短的正規表達式是可行的。以下是一個更簡潔的版本：

```
\S+@\S+\.\S+
```

- `\S+`：匹配一個或多個非空白字元。這將匹配 `@` 符號前後的任何字元序列。
- `@`：匹配 `@` 符號。
- `\S+`：匹配一個或多個非空白字元。這將匹配電子郵件地址中網域部分的任何字元序列。
- `\.`：匹配一個句點（點），用反斜線跳脫。
- `\S+`：匹配一個或多個非空白字元。這將匹配電子郵件地址中頂級網域 (TLD) 部分的任何字元序列。

這個較短的正規表達式模式也能擷取大多數有效的電子郵件地址，但就像較長的版本一樣，它可能無法涵蓋所有可能的邊界情況，或完全符合所有電子郵件地址標準。

## 8.3.1 搜尋和替換日期

在這個範例中，我們正在尋找一個 Python 腳本，用來替換文字檔中所有以 MM/DD/YYYY 和 DD.MM.YYYY 格式編寫的日期，轉換為 ISO 格式，即 YYYY-MM-DD。

**提示：** Python：我想將文字檔中的所有國際日期 (12/31/2024) 和德國日期 (31.12.2024) 替換為 ISO 日期 (2024-12-31)。

以下是一個 Python 腳本，它會讀取文字檔，將所有國際和德國日期格式替換為 ISO 日期格式，並將變更寫回檔案：

```python
import re

def convert_dates_to_iso(file_path):
    with open(file_path, 'r') as file:
        content = file.read()

    # 國際日期的模式 (MM/DD/YYYY)
    content = re.sub(r'(\b\d{1,2})/(\d{1,2})/(\d{4}\b)', 
                     r'\3-\1-\2', content)

    # 德國日期的模式 (DD.MM.YYYY)
    content = re.sub(r'(\b\d{1,2})\.(\d{1,2})\.(\d{4}\b)', 
                     r'\3-\2-\1', content)
    
    with open(file_path, 'w') as file:
        file.write(content)

# 範例用法
convert_dates_to_iso('path_to_your_file.txt')
```

這個解決方案值得注意的是群組的處理方式。群組可以在正規表達式中使用括號來定義。在第二個替換表達式中，`\1` 表示第一個群組的內容，`\2` 表示第二個群組的內容，以此類推。透過這種方式，可以從 `31.12.2024` 建立新的 `2024-12-31` 字串。

## 8.3.2 使用 grep 搜尋檔案

以下範例的起點是一個 HTML 檔案和一個包含圖片檔案的目錄。一方面，我們想要刪除所有未在 HTML 檔案中被連結的圖片檔案；另一方面，我們想要確保圖片檔案目錄是完整的，也就是說，HTML 檔案不會參照此目錄中根本不存在的圖片。

**提示：** 正規表達式/grep：我想從 HTML 檔案中擷取所有圖片檔案名稱。圖片檔案名稱總是以「screenshots/」開頭，以「.jpg」或「.png」結尾。能幫我嗎？

您可以將正規表達式與 grep 一起使用，從 HTML 檔案中擷取圖片檔案名稱。以下是您可以使用的命令：

```bash
grep -o 'screenshots/[^"]*\.jpg\|screenshots/[^"]*\.png' \
  your_html_file.html
```

說明：
- `-o` 選項：僅印出匹配行中的匹配部分。
- `'screenshots/[^"]*\.jpg\|screenshots/[^"]*\.png'`：這個正規表達式匹配以 `screenshots/` 開頭、後接除 `"` 以外的任何字元、並以 `.jpg` 或 `.png` 結尾的字串。

在後續問題中，我們請 ChatGPT 提供刪除不必要圖片的方法。結果是一個 Bash 腳本。

**提示：** 太好了。現在我想刪除所有不在 grep 結果中的圖片檔案。（screenshots 目錄包含多餘的圖片。）

您可以透過在 shell 腳本中結合 `grep` 和 `rm` 命令來實現。以下是應該能完成工作的腳本：

```bash
#!/bin/bash

# 包含圖片檔案的目錄
image_dir="screenshots"

# HTML 檔案
html_file="your_html_file.html"

# 使用 grep 查找所有圖片檔案名稱
image_files=$(grep -o \
  'screenshots/[^"]*\.jpg\|screenshots/[^"]*\.png' \
  "$html_file")

# 遍歷圖片目錄中的所有檔案
for file in "$image_dir"/*; do
    # 從完整路徑中提取檔案名稱
    filename=$(basename "$file")

    # 檢查檔案名稱是否不在圖片檔案列表中
    if ! grep -q "screenshots/$filename" <<< "$image_files"; 
    then
        # 如果未找到，刪除檔案
        rm "$file"
        echo "Deleted $file"
    fi
done
```

當然，只有在確定所有檔案都有備份之後，您才會嘗試這樣的腳本！幸運的是，這裡一切運作完美。

**提示：** 完美。現在我們需要反向操作。我需要另一個腳本，列出 HTML 檔案中參照但在 screenshots 目錄中缺少的圖片檔案。

```bash
#!/bin/bash

image_dir="screenshots"
html_file="your_html_file.html"

referenced_files=$(grep -o \
  'screenshots/[^"]*\.jpg\|screenshots/[^"]*\.png' \
  "$html_file")

# 遍歷 HTML 檔案中參照的所有檔案
for ref_file in $referenced_files; do
    # 從路徑中提取檔案名稱
    filename=$(basename "$ref_file")

    # 檢查檔案是否存在於圖片目錄中
    if [ ! -f "$image_dir/$filename" ]; then
        # 如果未找到，印出缺少的檔案
        echo "Missing file: $filename"
    fi
done
```
