# 11.2 使用 OpenHands

OpenHands 是一個開源的 AI 軟體工程師平台，提供了一個完整的環境，讓 AI 代理可以在其中自主運作。本節將探討如何設定和使用 OpenHands 進行 AI 輔助程式設計。

## 設定 OpenHands

### 先決條件

在安裝 OpenHands 之前，請確保您的系統符合以下要求：

- **Docker**：OpenHands 在容器化環境中運作，需要安裝 Docker
- **Python 3.12+**：後端執行所需
- **API 金鑰**：您需要一個相容 LLM 提供者的 API 金鑰（例如 OpenAI、Anthropic 等）

### 安裝步驟

1. **使用 Docker 安裝（推薦）**：

```bash
# 拉取並執行 OpenHands
docker run -it --rm \
  -p 3000:3000 \
  -v /var/run/docker.sock:/var/run/docker.sock \
  ghcr.io/all-hands-ai/openhands:latest
```

2. **使用 pip 安裝**：

```bash
pip install openhands
```

### 設定

安裝完成後，設定您的 LLM 提供者和 API 金鑰：

```bash
# 在設定介面中或透過環境變數設定
export LLM_API_KEY="your-api-key-here"
export LLM_MODEL="claude-3-5-sonnet-20241022"
```

## OpenHands 介面

OpenHands 透過網頁瀏覽器提供了一個視覺化介面，包括：

### 主要元件

1. **聊天面板**：您與 AI 代理溝通的地方
2. **工作區**：AI 代理運作的檔案系統視圖
3. **終端機**：命令的即時終端機輸出
4. **程式碼編輯器**：程式碼變更的即時視圖
5. **瀏覽器**：AI 代理使用的內建網頁瀏覽器

## 基本 usage

### 啟動工作階段

若要開始使用 OpenHands：

1. 在連接埠 3000 開啟 OpenHands（預設）
2. 在設定頁面輸入您的 API 金鑰
3. 建立新的工作區或開啟現有的工作區
4. 開始向 AI 代理描述您的任務

### 範例：建立一個簡單的應用程式

**提示：**

```
Create a Python Flask web application that:
1. Has a home page with a welcome message
2. Has an /api/status endpoint that returns JSON
3. Includes basic error handling
4. Has a requirements.txt file
5. Includes a README.md with setup instructions
```

OpenHands 代理將：
1. 規劃任務結構
2. 建立必要的檔案
3. 編寫程式碼
4. 安裝相依套件
5. 測試應用程式
6. 報告結果

### 範例：重構程式碼

**提示：**

```
I have a Python project in the workspace. Please:
1. Review the existing code structure
2. Identify any code smells or areas for improvement
3. Refactor the code following PEP 8 guidelines
4. Add type hints where missing
5. Add docstrings to all functions and classes
6. Run the existing tests to make sure nothing is broken
```

## 進階功能

### 使用現有儲存庫

您可以將現有的 Git 儲存庫匯入 OpenHands：

**提示：**

```
Clone the repository https://github.com/example/my-project.git 
and then analyze the codebase. Provide a summary of:
1. The overall architecture
2. Key components and their responsibilities  
3. Dependencies and their purposes
4. Any potential issues or improvements
```

### 執行測試

OpenHands 可以自主執行測試並修正問題：

**提示：**

```
Run all the tests in the project. If any tests fail:
1. Analyze the failure messages
2. Fix the underlying issues
3. Re-run the tests to verify the fixes
4. Report what was wrong and what was fixed
```

### 除錯

OpenHands 可以透過以下方式協助除錯：

- 讀取錯誤訊息和堆疊追蹤
- 分析相關程式碼
- 提出並實作修正
- 驗證修正是否解決了問題

**提示：**

```
I'm getting the following error when running my application:

```
Traceback (most recent call last):
  File "app.py", line 42, in <module>
    result = process_data(data)
  File "app.py", line 28, in process_data
    return items[key]
KeyError: 'missing_key'
```

Please analyze this error, find the root cause, and fix it.
```

## 最佳實踐

1. **要具體**：向 AI 代理提供明確、詳細的指示
2. **逐步驗證**：檢查 AI 代理在每個主要步驟後的工作
3. **使用版本控制**：定期提交變更，以便在需要時回復
4. **提供背景資訊**：分享相關檔案、文件或需求
5. **適當地分解任務**：將複雜任務分解為更小、易於管理的部分
6. **檢閱生成的程式碼**：始終檢閱 AI 生成的程式碼是否正確性和安全性

## 限制與考量

- **資源使用**：OpenHands 需要大量的計算資源
- **API 成本**：自主代理可能會快速消耗 API 點數
- **安全性**：儘管在沙盒中運作，仍要檢閱生成的程式碼是否存在安全問題
- **學習曲線**：有效地撰寫提示需要練習
- **除錯 AI 行為**：理解 AI 代理為何採取特定行動可能具挑戰性

## 總結

OpenHands 為 AI 輔助程式設計提供了一個強大的自主環境。透過了解其功能並遵循最佳實踐，開發者可以利用這個工具來處理複雜的程式設計任務，同時保持對開發過程的控制。其視覺化介面和全面的工具整合使其成為 AI 輔助開發工作流程中的寶貴資產。
