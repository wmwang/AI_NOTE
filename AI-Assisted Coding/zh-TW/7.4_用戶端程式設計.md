# 7.4 用戶端程式設計

資料庫設計和 DBMS 管理通常所需的時間遠少於開發用戶端程式碼，無論是網頁介面、應用程式，還是位於 DBMS 和實際用戶端程式碼之間的商業類別。當然，AI 工具在這種情況下也能協助編寫程式碼。

根據我們的經驗，AI 聊天僅在設計階段有用，也就是當您選擇合適的資料庫函式庫或開發和測試初始函式時。因此，對於詳細的問題，直接使用 AI 輔助編輯器來修改程式碼會更有效率。您只需要使用一個像 `getRecentOrders` 這樣的函式名稱，GitHub Copilot 或其他 AI 工具就會建議合適的程式碼。程式碼庫中可作為參考的類似函式或方法越多，效果就越好。

我們在本節中不印出較長的程式碼範例，而僅提供一些提示建議。獲得有意義回答的基本前提是 AI 工具能夠存取資料庫結構描述。然而，我們發現許多 AI 工具即使有結構描述也無法正確擷取欄位名稱。有時候，在提示中以 CREATE TABLE 命令的形式僅指定相關的資料表會有所幫助。

**提示：** PHP/MySQL/MySQLi：我需要一份過去六個月內下過訂單的所有客戶的字母排序列表。結果應顯示在六欄的 HTML 表格中。每個結果都是一個連到詳細頁面 `customer?id=xxx` 的連結。

**提示：** Node.js/MySQL：我想寫一個函式，透過 REST 請求（GET `customers/lastOrderSince=yyyyy-mm-dd`）回傳客戶列表（如上所述）。

**提示：** Python/PyMySQL：我想向所有超過六個月未下訂單的客戶發送促銷優惠。你能提供一個指令碼嗎？

**提示：** PHP/MySQLi：如何將多個 SQL 命令捆绑成一個交易？

**提示：** C#：我應該使用哪個函式庫來存取 SQLite 資料庫？

**提示：** Bash：我在 mydbs.txt 中有一份 MariaDB 資料庫列表。我需要對所有資料庫進行結構描述更新。必要的 ALTER TABLE 命令在 v32.sql 中。你能提供一個指令碼嗎？
