# 2.4 範例：問答遊戲

在這個較為完整的範例中，我們假設您想要程式設計一個簡單的問答遊戲。程式從題庫中隨機選取幾道問題，然後提出問題並提供多個選擇題答案，以字母 a、b、c 等標記供玩家選擇。玩家輸入一個字串（例如 `ad`，表示第一個和第四個答案正確），程式分析答案後再提出下一道問題。

## 2.4.1 類別設計

本節主要關注程式所需類別的組織。您在類別設計方面經驗不足，因此向 AI 工具尋求幫助。在第一個非常詳細的提示中，您概述了程式的基本功能並指定了程式語言。您立即向 AI 工具說明這是多選題，允許多個正確答案。這對程式設計來說是重要的資訊。沒有這個資訊，ChatGPT 很可能會假設每道題只有一個正確答案。如果您在此時以這個假設開發程式，之後擴展功能時將需要進行大量修改。最好現在就釐清這個細節。

**提示：** I would like to write a small quiz game in Java. There is a predefined pool of questions. When the program starts, the player has to answer five questions randomly chosen from the pool. For each question, there is a set of possible answers (min. 2, max. 6). At least one answer has to be correct. Some questions may have more than one correct answer. In this case, ALL correct answers must be selected. Please help me design the necessary classes. Don't provide any finished code yet. Just help me decide on good class names and define the right instance variables. Leave constructors and methods for later.

> 在 Java 中設計問答遊戲需要清晰且結構良好的類別來管理問題、答案和遊戲流程。以下是建議的類別設計及其實例變數：
>
> - **QuizGame：** 管理整體遊戲流程和邏輯。實例變數：
>   - `List<Question> questionPool`：存放遊戲中所有可能的問題。
>   - `List<Question> selectedQuestions`：儲存目前遊戲中隨機選取的問題。
>   - `int score`：追蹤玩家的分數。
>   - `int currentQuestionIndex`：追蹤目前遊戲中的問題索引。
>
> - **Question：** 代表一道問答題。實例變數：
>   - `String text`：問題的文字。
>   - `List<Answer> answers`：這道題的可能答案列表。
>   - `List<Answer> correctAnswers`：正確答案列表。
>   - `int id`：問題的唯一識別碼（選填，方便管理）。
>
> - **Answer：** 代表問答題的一個可能答案。實例變數：
>   - `String text`：答案的文字。
>   - `boolean isCorrect`：指示此答案是否正確。
>   - `int id`：答案的唯一識別碼（選填，方便管理）。
>
> - **Player：** 代表遊戲中的玩家 [...]。
> - **QuizUtil：** 用於輔助方法的工具類別（例如隨機選取、驗證...）。
> [詳細描述]

這個建議絕對有用。您決定最初將程式碼組織為三個類別：Main、Question 和 Answer。您開始獨立實作，在 Main 中設定起始程式碼，並程式設計簡單的 Answer 類別。

## 2.4.2 實作 Question 類別

在下一個提示中，您要求在程式設計 Question 類別方面獲得支援。重要的是告訴您的 AI 工具已經存在哪些程式碼，因為它必須在 Question 類別中參考它。

**提示：** OK. I created a Main class to run/test the program. I also wrote the Answer class like this:
```java
public class Answer {
    public String text;
    boolean isCorrect;
    // constructor
    public Answer(String text, boolean isCorrect) {
        this.text = text;
        this.isCorrect = isCorrect;
    }
}
```
Next, I want to implement Question. Please suggest initial code (only instance variables and the constructor, without the optional id variable).

```java
import ...;
public class Question {
    public String text;
    public List<Answer> answers;
    public List<Answer> correctAnswers;
    
    // Constructor
    public Question(String text,
                    List<Answer> answers,
                    List<Answer> correctAnswers) {
        this.text = text;
        this.answers = answers;
        this.correctAnswers = correctAnswers;
    }
}
```

## 2.4.3 用於測試的題庫

在查看這些類別的其他方法之前，您需要測試材料，即一些問答題。建立它們是一項相當枯燥的任務，可以委派給 ChatGPT 等工具。

**提示：** As a base for testing the code, please provide a method for the Main class to initialize a pool of 10 geographic questions.

ChatGPT 最初只提供了恰好一個正確答案的問題。AI 工具只有在被要求時才會新增具有多個匹配答案的問題。

**提示：** Perfect. However, you could include a few questions with more than one correct answer (e.g., rivers in Brazil)?

## 2.4.4 Question 類別的方法

現在我們有了第一批問答題，現在是設計 Question 類別的時候了。這裡至少需要兩個方法：一個用於顯示問題及其解決方案並輸入答案，第二個用於評估答案。在以下提示中，您要求實作第一個方法，相對精確地指定問題應如何呈現（答案應以小寫字母標記）以及程式期望答案的格式。為了防止 AI 工具輸出整個類別的程式碼，我們要求它只回傳新方法。

**提示：** Add a method to the Question class to print the question text and all of the enumerated answers such as a), b), c), and so on. The method prompts the user for an answer string, for example, "ad" if answers a) and d) are correct. Provide code only for the method, not for the entire class.

AI 工具建議了兩個新方法：`askQuestion` 實作了提示的規範並將答案作為字串回傳。未經要求的 `validateAnswer` 方法評估此字串並檢查它是否包含所有適用的答案，但不包含不適用答案的進一步字母。

在這個範例中，`validateAnswer` 方法是最棘手的。但在您甚至來不及思考如何自己程式設計字串評估之前，ChatGPT 已經提供了一個現成的方法。這令人煩惱。

到目前為止建立的程式碼可以在本書的範例檔案中找到。此時，我們離開這個範例。當然，完成程式還缺少各種方法（遊戲流程、隨機選取五道問題、問題呈現、結果顯示[例如，五道問題中答對了三道]等）。也許您想自己完成這個範例——這將是使用 AI 工具的一個好練習。或者，我們能否用這個範例激勵您在 AI 的幫助下自己實作這個或類似的遊戲？您決定自己想要在多大程度上獲得支援。

> **過度熱心的學習輔助工具**
> 遺憾的是，ChatGPT 等工具在作為學習輔助工具使用時往往過於熱心。AI 工具經常提供比必要的更多的程式碼（即使您在提示中明確要求相反）。一旦程式碼出現在螢幕上並且您已經閱讀了它，您就已經受到了影響。
>
> 當然，沒有人強迫您採用程式碼，但誘惑很大。如果您自己開發更多方法，學習效果會更大。如果您確實在問題上失敗了，您隨時可以為您半完成的程式碼要求改進建議。

## 2.4.6 AI 機器人還是程式碼精靈？

您的程式碼越來越廣泛，您委派給 AI 工具的工作越多，使用像 ChatGPT 這樣的聊天系統或類似工具就越不合適。問題在於，如果您自己開發了部分程式碼，AI 工具會失去對程式碼的追蹤。

當然，您可以不時地將程式的目前狀態插入提示中，或透過拖放使其可用。隨著類別數量的增加，您可以透過使用對整個程式碼具有永久存取權的程式碼精靈來避免這種越來越繁瑣的來回操作。

反對使用程式碼精靈的一個理由是，它不僅在明確要求時才會啟動，而且在每次輸入或更改程式碼時都會提出新增建議。這對於高效地撰寫程式碼是有用的，但對於學習則不太適合。
